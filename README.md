# sunshengyao-learning

> 我的个人学习仓库 · 基于 Bloom 掌握学习法 · AI 导师根据我的背景和反馈动态调整教学内容

这是一个**我的个人学习记录仓库**，配合 AI 导师（MyFlicker）为我提供个性化学习路径。

- **客户端方向**：建立 Android / iOS / 鸿蒙 三端的知识体系
- **前端方向**：建立 React / Vue / 小程序 / H5 四端的知识体系

> 两个方向互不串通：同方向内跨端走对照模式；跨方向从入门走起，已有另一方向经验仅作为背景类比。

---

## ✨ 它是什么

- **不是** 视频课程、不是文档教程、不是 RAG 知识库
- **是** 一个 AI 导师的执行规范 + 我个人的学习记录
- AI 会根据我的背景、节奏、反馈，**为我一篇一篇生成**学习文章；我写反馈，它读反馈，迭代教学

核心特性：

| 特性 | 说明 |
|---|---|
| 三层类比教学 | 用我已知的知识（其他端 / 编程语言 / CS 通识）做类比锚点 |
| 双模学习路径 | 零基础入门 + 跨端迁移，自动判断 |
| 反馈驱动迭代 | 我不写反馈，AI 不会推进；写得越多，下一篇越精准 |
| 进度可视化 | 全流程在 git 仓库内，随时可回顾 |

## 📚 日常学习节奏

```
读 → 写反馈 → commit → "继续"
```

具体来说：

1. AI 生成 `sunshengyao/{端}/{分类}/01.md`（如 `sunshengyao/iOS/basics/01.md` 或 `sunshengyao/iOS/crash/01-zombie.md`）
2. 我打开文件，**仔细阅读正文**
3. 滚动到文末**反馈区**，填写：
   - **【我理解到的核心】**（必填，2-3 句话复述本文要点）
   - 还有疑问的地方（可选）
4. `git add . && git commit -m "feat: complete iOS/basics/01" && git push`
5. 回到 AI 对话，说"继续"或"下一篇"，AI 会读我的反馈，生成 `02.md`

> ⚠️ **重要约束**：如果我没填写"我理解到的核心"，AI 会**拒绝生成下一篇**。这是 Bloom 掌握学习法的核心机制 —— 没有真正的内化，就没有下一步。

## 🎯 学习阶段说明

整个学习路径按 **3 个阶段** 推进，每个阶段聚焦不同业务能力。两个方向各有三个阶段，命名相同但**涵盖概念不同**。每篇 md 末尾都会自动显示我当前的阶段进度。

### 客户端方向

| 阶段 | 名称 | 涵盖概念 | 完成后能做 |
|---|---|---|---|
| **阶段 1** | 入门（Onboarding） | UI 组件、生命周期、事件、导航 | 独立完成纯 UI 改造；可作为复杂需求里的"页面骨架"贡献者 |
| **阶段 2** | 进阶（Independent） | 状态管理、网络、持久化、异步 | 独立承接列表页、详情页、表单、登录注册等带数据流转的常规业务需求 |
| **阶段 3** | 完备（Production-Ready） | 权限、资源/国际化、综合训练 | 交付符合上线标准的功能；具备该端正式 owner 资格 |

### 前端方向

| 阶段 | 名称 | 涵盖概念 | 完成后能做 |
|---|---|---|---|
| **阶段 1** | 入门（Onboarding） | 组件模型与 JSX/模板、生命周期与 Hooks、事件、路由 | 独立完成静态页面、营销活动落地页、设计稿还原 |
| **阶段 2** | 进阶（Independent） | 状态管理、网络请求、浏览器存储、异步与 Promise | 独立承接列表页、详情页、表单、本地缓存等常规业务 |
| **阶段 3** | 完备（Production-Ready） | 性能与渲染优化、工程化、综合训练 | 交付含性能调优、可监控可回滚的上线需求；具备该端正式 owner 资格 |

**阶段判定规则**（两方向一致）：

- 该阶段所有概念都 ✅ → 阶段已完成
- 至少 1 个 ✅ 或 ⏳ → 阶段进行中
- 全部 ⚪ → 阶段未开始

每篇 md 末尾的"📊 学习进度"小节会显示当前阶段、阶段内已完成/总概念数、当前能力等级、下一里程碑。

> 同方向内跨端学习（如 Android → iOS）通常通过"对照模式"在 2-4 周内完成全部 3 个阶段。跨方向（如客户端 → 前端）需从阶段 1 重新走起，另一方向经验仅作为背景类比。

## ✍️ 反馈写作指南

### 必填项：我理解到的核心

不是抄文章原话，而是**用自己的话复述**。建议 2-3 句，覆盖：

- 这篇文章的核心概念是什么
- 它和我已经知道的东西怎么联系
- 关键的 1-2 个要点

**好例子**：
> Activity 是 Android 单个屏幕的载体，类似 iOS 的 UIViewController 或者 Web 的一个页面。它有 7 个生命周期回调，最关键的是 onCreate（初始化）和 onDestroy（清理资源），中间的 onPause/onResume 处理可见性。

**差例子**：
> 学完了，理解了。

### 可选项：还有疑问的地方

写下卡点，AI 下一篇会优先解决。例如：

> 不太清楚为什么 onPause 和 onStop 是分开的，什么时候会只触发 onPause 不触发 onStop？

## ❓ FAQ

### Q1：我想换一端学，或者换个方向学，怎么操作？

**同方向内换端**（如 Android → iOS）：直接在对话里告诉 AI：

> 我想开始学 iOS 了

AI 会读 profile.md，自动判断该用入门模式还是跨端过渡模式（如果我 Android 已掌握一些公共概念，会在 iOS 端使用对照模式加速）。

**跨方向**（如客户端 → 前端）：

- 初始选了"都想学"：在主方向完成阶段 1 后，说"我想开始学 {二方向}"，AI 会创建二方向目录并从阶段 1 走起
- 初始只选了一个方向但后面想加：可以让 AI 帮我更新 profile.md 的开发方向字段为"客户端 + 前端待开启"，仍然需主方向完成阶段 1 后才能开启二方向

> ⚠️ 跨方向学习不会拿另一方向的 ✅ 作为预填。例如我是 Android 高手，学 React 时 React 方向依然从零起步，Android 的背景仅作为类比加速理解。

### Q2：我的 md 文件越来越长，反馈区找不到了？

这是正常的。建议：

- 一篇文章的字数控制在 2000-4000 字（AI 默认会控制）
- 单个公共概念如果太大（如"UI 组件体系"），AI 会自动拆成多篇（`01.md`、`02.md`、`03.md`）
- 反馈区永远在文件最末尾，按 `Cmd+End` / `Ctrl+End` 直接跳到底部

### Q3：我中断了一段时间，回来怎么继续？

直接打开 AI 对话说"继续"。AI 会读 profile.md 中的"最近学习时间"，如果超过 30 天会建议我先复习关键概念，再进入下一篇。

### Q4：我可以同时学两端吗？

**同方向内两端**（如 Android + iOS）：技术上可以，但不推荐。AI 会建议我先完成一端的至少 5 个核心概念，再开第二端，否则类比锚点不够强。

**同时学两个方向**（如客户端 + 前端）：**不允许并行**。需要先完成主方向阶段 1，才能开启二方向。这是为避免精力过度分散导致两边都不扞实。
