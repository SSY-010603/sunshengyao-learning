# iOS 开发入门：UI 组件体系

> **本文覆盖的公共概念**：UI 组件体系

## 从你熟悉的地方开始

你写过 C++，对面向对象应该不陌生。在 C++ 里，你可能会用一个类来表示一个"按钮"，它有成员变量（标题、颜色）和成员函数（点击响应）。iOS 的 UI 组件体系也是类似的思路——**用对象来描述界面元素，用树状结构来组织它们的关系**。

只不过，iOS 把这件事做得更优雅、更系统化。

---

## 核心概念：UIView —— 一切 UI 的基类

iOS 的界面由一个个 **UIView** 组成。你可以把 UIView 理解为 C++ 中的一个基类：

```cpp
// C++ 中你可能会这样想象它
class View {
    Rect frame;       // 位置和大小
    Color backgroundColor;  // 背景色
    vector<View*> subviews; // 子视图列表
    View* superview;        // 父视图
};
```

在 iOS 中，UIView 是真实的 UIKit 基类，所有你能看到的界面元素——按钮、标签、图片、滚动列表——最终都继承自 UIView。

**关键心智模型**：UIView 是一棵树。每个 View 可以包含多个子 View（subviews），每个子 View 只有一个父 View（superview）。这棵树就叫 **视图层级（View Hierarchy）**。

```
UIWindow
  └── UIViewController.view (根视图)
       ├── UINavigationBar
       │    ├── UILabel (标题)
       │    └── UIButton (返回按钮)
       └── UIView (内容区域)
            ├── UIImageView (头像)
            ├── UILabel (用户名)
            └── UIButton (关注按钮)
```

这就像 C++ 中的组合模式（Composite Pattern）：一个对象可以包含同类型的子对象，形成树状结构。你在 C++ 里设计过嵌套数据结构的话，这个概念应该很好理解。

## 基础控件：UIKit 的"积木"

UIKit 提供了一系列现成的 UI 组件，就像 C++ 标准库提供现成的容器一样。以下是最常用的几个：

| 控件 | 类名 | 作用 | C++ 类比 |
|------|------|------|----------|
| 标签 | UILabel | 显示不可编辑的文字 | 一个只读的 string 成员 |
| 按钮 | UIButton | 接受点击，触发动作 | 带回调的 Button 类 |
| 图片 | UIImageView | 显示图片 | 一个封装了像素数据的显示类 |
| 文本框 | UITextField | 单行文本输入 | cin 的 GUI 版本 |
| 多行文本 | UITextView | 多行文本编辑 | 一个富文本编辑器控件 |
| 滚动视图 | UIScrollView | 可滚动的内容区域 | 带滚动条的自定义容器 |
| 列表 | UITableView | 展示可滚动的列表 | 最常用的列表控件 |
| 堆栈 | UIStackView | 自动排列子视图 | 类似 Flexbox 的自动布局容器 |

> **特别注意**：iOS 有两套 UI 框架——**UIKit**（命令式，用 Swift/ObjC 命令式地创建和操作视图）和 **SwiftUI**（声明式，用声明式语法描述界面）。你现在学的是 UIKit，因为它是 iOS 开发的基础，理解 UIKit 后再学 SwiftUI 会更顺畅。两者之间的关系类似 C++ 中"手动管理内存"与"智能指针"——前者让你理解底层机制，后者让你写得更简洁。

## 第一个实战：用代码创建一个简单界面

既然你已经接触过 iOS 的基础语法，我们来用 Swift + UIKit 写一个最简单的界面：

```swift
import UIKit

class ViewController: UIViewController {
    
    override func viewDidLoad() {
        super.viewDidLoad()
        
        // 1. 设置背景色
        view.backgroundColor = .white
        
        // 2. 创建一个标签
        let titleLabel = UILabel()
        titleLabel.text = "Hello, iOS!"
        titleLabel.textColor = .black
        titleLabel.font = UIFont.systemFont(ofSize: 24, weight: .bold)
        titleLabel.textAlignment = .center
        
        // 3. 创建一个按钮
        let actionButton = UIButton(type: .system)
        actionButton.setTitle("点我", for: .normal)
        actionButton.backgroundColor = UIColor.systemBlue
        actionButton.setTitleColor(.white, for: .normal)
        actionButton.layer.cornerRadius = 8
        
        // 4. 按钮点击事件
        actionButton.addTarget(self, 
                               action: #selector(buttonTapped), 
                               for: .touchUpInside)
        
        // 5. 添加到视图层级
        view.addSubview(titleLabel)
        view.addSubview(actionButton)
        
        // 6. 设置布局（用 Auto Layout）
        titleLabel.translatesAutoresizingMaskIntoConstraints = false
        actionButton.translatesAutoresizingMaskIntoConstraints = false
        
        NSLayoutConstraint.activate([
            titleLabel.centerXAnchor.constraint(equalTo: view.centerXAnchor),
            titleLabel.topAnchor.constraint(equalTo: view.safeAreaLayoutGuide.topAnchor, 
                                             constant: 100),
            
            actionButton.centerXAnchor.constraint(equalTo: view.centerXAnchor),
            actionButton.topAnchor.constraint(equalTo: titleLabel.bottomAnchor, 
                                              constant: 20),
            actionButton.widthAnchor.constraint(equalToConstant: 120),
            actionButton.heightAnchor.constraint(equalToConstant: 44)
        ])
    }
    
    @objc func buttonTapped() {
        print("按钮被点击了！")
    }
}
```

### 逐段解析

**第 1 步**：`view.backgroundColor` — `view` 是 UIViewController 自带的根视图，你给它设个颜色，就有了最基本的界面。

**第 2-3 步**：创建 UILabel 和 UIButton，设置它们的属性。这就像在 C++ 中构造一个对象并初始化其成员变量。

**第 4 步**：`addTarget(action:for:)` — 这是 UIKit 的事件绑定方式。Target-Action 模式是 iOS 特有的设计模式，你可以理解为 C++ 中的函数回调注册：当按钮被点击（`.touchUpInside` 事件）时，调用 `buttonTapped` 方法。

**第 5 步**：`addSubview` — 把子视图添加到父视图上，建立视图层级。这一步是必须的，否则创建的控件不会显示。就像在 C++ 中，你构造了一个对象但没把它放进容器，它就"不在场景中"。

**第 6 步**：Auto Layout — iOS 的布局系统。`translatesAutoresizingMaskIntoConstraints = false` 告诉系统"我要用约束来布局，别用老的 frame 方式"。然后用 `NSLayoutConstraint.activate` 设置约束——这就相当于告诉系统"标签要水平居中、距离顶部 100 点"。

> **C++ 程序员的直觉**：你可能会觉得 Auto Layout 比直接写 frame 坐标麻烦。但 Auto Layout 的好处是——当屏幕旋转、设备尺寸变化时，约束会自动调整，你不用手动计算每个控件的坐标。这就像用智能指针代替手动 new/delete：写起来多一步，但运行时更安全。

## UIStackView：让布局更简单

如果你觉得 Auto Layout 的约束写法太啰嗦，iOS 提供了一个更方便的布局容器——**UIStackView**：

```swift
let stackView = UIStackView()
stackView.axis = .vertical      // 垂直排列
stackView.spacing = 16          // 子视图间距
stackView.alignment = .center   // 子视图对齐方式

// 直接往 stackView 里加子视图，它会自动布局
stackView.addArrangedSubview(titleLabel)
stackView.addArrangedSubview(actionButton)

view.addSubview(stackView)

// StackView 本身需要约束，但子视图不需要了！
stackView.translatesAutoresizingMaskIntoConstraints = false
NSLayoutConstraint.activate([
    stackView.centerXAnchor.constraint(equalTo: view.centerXAnchor),
    stackView.centerYAnchor.constraint(equalTo: view.centerYAnchor)
])
```

UIStackView 就像一个自动排列的容器——你把子视图丢进去，它帮你搞定布局。类比到 C++，这就像你用一个 `vector` 来管理一组对象，`vector` 负责内存布局，你只需要 `push_back`。

## 声明式 vs 命令式：两种 UI 范式

你刚才写的 UIKit 代码是**命令式**的：你一步一步"命令"系统创建视图、设置属性、添加约束。这是 C++ 程序员很熟悉的风格——先声明对象，再初始化，再使用。

iOS 还有另一种方式——**SwiftUI**，它是声明式的：

```swift
// SwiftUI 写法（仅作对比，后续课程再深入）
var body: some View {
    VStack(spacing: 16) {
        Text("Hello, iOS!")
            .font(.title)
            .fontWeight(.bold)
        
        Button("点我") {
            print("按钮被点击了！")
        }
        .padding()
        .background(Color.blue)
        .foregroundColor(.white)
        .cornerRadius(8)
    }
}
```

声明式的核心区别：**你描述"界面应该长什么样"，而不是"怎么一步步构建界面"**。系统根据你的描述自动创建和管理视图。

| 维度 | 命令式（UIKit） | 声明式（SwiftUI） |
|------|----------------|-------------------|
| 思维方式 | "怎么做" — 逐步命令 | "是什么" — 描述结果 |
| 类比 | 手动管理内存 (new/delete) | 智能指针 (shared_ptr) |
| 布局方式 | Auto Layout 约束 | 声明式容器 (VStack/HStack) |
| 状态更新 | 手动修改视图属性 | 修改状态，UI 自动刷新 |
| 上手难度 | 中等，概念清晰 | 入门快，但调试需要经验 |

我们目前的学习路线以 **UIKit** 为主线，因为：
1. 你在工作中接触的 iOS 代码大概率是 UIKit（存量项目基本都是）
2. UIKit 让你理解 iOS UI 的底层机制
3. 学完 UIKit 再转 SwiftUI 会事半功倍

## 关键要点总结

1. **UIView 是一切 UI 的基类**，视图层级是一棵树，每个 View 可包含子视图（subviews）
2. **UIKit 提供了丰富的现成控件**：UILabel、UIButton、UIImageView、UITableView 等
3. **布局用 Auto Layout**：通过约束描述视图间的位置关系，自动适配不同屏幕
4. **UIStackView 简化布局**：自动排列子视图，减少手动约束代码
5. **命令式 vs 声明式**：UIKit 是命令式，SwiftUI 是声明式；先学 UIKit 打基础
