## 核心概念：触摸事件的三阶段

用户手指触碰屏幕的那一刻，iOS 会生成一个 **UITouch** 对象，封装了这次触摸的所有信息（位置、时间、阶段）。一次完整的触摸经历三个阶段：

```
手指触碰屏幕 → UITouch.phase = .began     （touchBegan）
手指在屏幕移动 → UITouch.phase = .moved     （touchMoved）
手指离开屏幕 → UITouch.phase = .ended       （touchEnded）
手指滑出区域   → UITouch.phase = .cancelled  （touchCancelled）
```

这就像 C++ 中处理输入事件：键盘按下（keyDown）、持续按着（keyRepeat）、松开（keyUp）。iOS 的触摸事件本质上也是一套状态机。

### UIResponder —— 事件响应的基类

所有能响应事件的对象都继承自 **UIResponder**。UIView 和 UIViewController 都是 UIResponder 的子类。UIResponder 提供了四个触摸回调：

```swift
// 在 UIView 或 UIViewController 的子类中重写
override func touchesBegan(_ touches: Set<UITouch>, with event: UIEvent?) {
    // 手指触碰
}

override func touchesMoved(_ touches: Set<UITouch>, with event: UIEvent?) {
    // 手指移动
}

override func touchesEnded(_ touches: Set<UITouch>, with event: UIEvent?) {
    // 手指抬起
}

override func touchesCancelled(_ touches: Set<UITouch>, with event: UIEvent?) {
    // 事件被中断（如来电）
}
```

> **C++ 类比**：这就像 C++ 中的虚函数回调机制。基类定义了 `onTouchBegan()` 等虚函数，子类通过重写来处理事件。UIKit 帮你做好了分发，你只需要在对应的回调里写逻辑。

## 事件分发机制：hitTest 与响应链

这是 iOS 事件处理中最核心的机制，也是面试高频题。分两个阶段理解：

### 阶段一：命中测试（Hit Testing）—— 找"谁被点了"

当用户触摸屏幕时，系统需要确定**哪个视图被点中了**。它从最顶层的窗口开始，沿着视图层级向下搜索：

```
1. 从 UIWindow 开始，调用 hitTest(_:with:)
2. 检查自身：是否可交互(userInteractionEnabled)、是否透明(alpha>0.01)、是否在范围内
3. 从后往前遍历 subviews（最后添加的在最上面，优先检查）
4. 对每个子视图递归调用 hitTest
5. 返回最深层被点中的视图，或返回 nil
```

**关键代码**：

```swift
// UIView 的 hitTest 默认实现（伪代码，帮你理解逻辑）
func hitTest(_ point: CGPoint, with event: UIEvent?) -> UIView? {
    // 1. 自己能响应吗？
    if !isUserInteractionEnabled || alpha < 0.01 || isHidden {
        return nil
    }
    // 2. 点在自己的范围内吗？
    if !point(inside: point, with: event) {
        return nil
    }
    // 3. 从后往前遍历子视图（最后添加的在最上面）
    for subview in subviews.reversed() {
        let convertedPoint = subview.convert(point, from: self)
        if let hitView = subview.hitTest(convertedPoint, with: event) {
            return hitView  // 找到了！返回最深的被点中的视图
        }
    }
    // 4. 子视图都没命中，那就是自己被点了
    return self
}
```

> **C++ 直觉**：这就像 C++ 中多态的动态分发——系统通过虚函数表找到"最具体的"实现。hitTest 也是在找"最具体的"被点中的视图（最深层的子视图）。

### 阶段二：响应链（Responder Chain）—— 找"谁来处理"

命中测试找到了被点中的视图后，事件会沿**响应链**向上传递，直到有人处理它：

```
被点中的 UIView
    │
    ▼ 没处理？传给 superview
superview (UIView)
    │
    ▼ 没处理？传给 UIViewController
UIViewController
    │
    ▼ 没处理？传给 UIWindow
UIWindow
    │
    ▼ 没处理？传给 UIApplication
UIApplication
    │
    ▼ 没处理？丢弃
App Delegate
```

```swift
// 响应链示例：子视图不处理，事件冒泡到父视图
class ParentView: UIView {
    override func touchesBegan(_ touches: Set<UITouch>, with event: UIEvent?) {
        print("ParentView 收到触摸事件")
    }
}

class ChildView: UIView {
    override func touchesBegan(_ touches: Set<UITouch>, with event: UIEvent?) {
        // 不调用 super，事件到此为止
        print("ChildView 处理了事件，不再向上传递")
    }
}
```

> **C++ 操作系统类比**：命中测试就像中断向量表——确定"哪个设备产生了中断"；响应链就像中断处理函数的调用栈——从最具体的处理器开始，逐级向上回退，直到有人处理。

## UIControl 与 Target-Action 模式

在 01.md 中你用过 `addTarget(action:for:)`，这是 UIKit 提供的高层事件接口。UIButton、UISlider、UISwitch 等控件都继承自 **UIControl**，它封装了触摸事件，提供更友好的事件绑定：

```swift
// UIControl 事件类型（常用的几个）
button.addTarget(self, action: #selector(tapped), for: .touchUpInside)       // 点击抬起
button.addTarget(self, action: #selector(pressed), for: .touchDown)          // 按下
button.addTarget(self, action: #selector(dragExit), for: .touchDragExit)     // 拖出按钮区域
slider.addTarget(self, action: #selector(valueChanged), for: .valueChanged)  // 值改变
switchControl.addTarget(self, action: #selector(toggled), for: .valueChanged)
```

### Target-Action 的工作原理

```
用户点击按钮
    │
    ▼
UIControl 内部处理触摸事件
    │
    ▼
判断事件类型（touchUpInside? touchDown? ...）
    │
    ▼
查找注册的 target-action 对
    │
    ▼
调用 target.action()  →  你的代码执行
```

**与 touchesBegan 的关系**：UIControl 在底层使用 touchesBegan/Ended 来检测触摸，然后将其"翻译"成语义化的事件（如 `.touchUpInside`），再通过 Target-Action 分发给你。你不需要直接处理 touches 回调——UIControl 帮你做了。

```
底层：touchesBegan → touchesEnded
         │
         ▼ UIControl 封装
高层：.touchUpInside → addTarget(action:for:) → 你的代码
```

> **C++ 类比**：这就像 C++ 中的信号-槽机制（Qt）或回调函数注册。底层是操作系统的中断/输入事件，上层框架将其封装为语义化的信号（clicked、valueChanged），你只需要 `connect` 对应的槽函数。

## 手势识别：UIGestureRecognizer

除了按钮点击，iOS 还有更丰富的手势交互——捏合缩放、左滑返回、长按等。这些由 **UIGestureRecognizer**（手势识别器）处理。

### 常用内置手势

| 手势 | 类名 | 用途 |
|------|------|------|
| 点击 | UITapGestureRecognizer | 单击、双击 |
| 长按 | UILongPressGestureRecognizer | 长按触发菜单 |
| 拖拽 | UIPanGestureRecognizer | 拖动视图 |
| 捏合 | UIPinchGestureRecognizer | 缩放图片 |
| 旋转 | UIRotationGestureRecognizer | 旋转对象 |
| 轻扫 | UISwipeGestureRecognizer | 左右上下滑动 |
| 屏幕边缘 | UIScreenEdgePanGestureRecognizer | 边缘滑动返回 |

### 使用方式

```swift
// 给视图添加点击手势
let tapGesture = UITapGestureRecognizer(target: self, 
                                         action: #selector(handleTap(_:)))
imageView.addGestureRecognizer(tapGesture)
imageView.isUserInteractionEnabled = true  // ⚠️ UIImageView 默认不响应事件，必须开启！

// 给视图添加捏合手势
let pinchGesture = UIPinchGestureRecognizer(target: self, 
                                             action: #selector(handlePinch(_:)))
imageView.addGestureRecognizer(pinchGesture)

// 处理手势
@objc func handleTap(_ gesture: UITapGestureRecognizer) {
    if gesture.state == .ended {
        let location = gesture.location(in: imageView)
        print("点击位置：\(location)")
    }
}

@objc func handlePinch(_ gesture: UIPinchGestureRecognizer) {
    switch gesture.state {
    case .began:
        print("开始捏合")
    case .changed:
        let scale = gesture.scale
        imageView.transform = CGAffineTransform(scaleX: scale, y: scale)
    case .ended:
        print("捏合结束，最终缩放：\(gesture.scale)")
    default:
        break
    }
}
```

### 手势冲突处理

当多个手势同时触发时（比如捏合和旋转），需要处理冲突：

```swift
// 要求某个手势必须在另一个手势失败后才识别
tapGesture.require(toFail: doubleTapGesture)
// 这样：单击不会立即触发，先等双击失败
```

```swift
// 实现 UIGestureRecognizerDelegate 来控制是否允许同时识别
func gestureRecognizer(_ gestureRecognizer: UIGestureRecognizer, 
                       shouldRecognizeSimultaneouslyWith other: UIGestureRecognizer) -> Bool {
    return true  // 允许同时识别（如捏合+旋转）
}
```

## 实战：一个可拖拽的图片卡片

把前面学过的知识串起来——UI 组件 + 生命周期 + 事件处理：

```swift
import UIKit

class DraggableCardViewController: UIViewController {
    
    let cardView = UIView()
    let imageView = UIImageView()
    let titleLabel = UILabel()
    
    // 记录拖拽起始位置
    var cardOriginalCenter: CGPoint = .zero
    
    override func viewDidLoad() {
        super.viewDidLoad()
        view.backgroundColor = .white
        
        // 卡片容器
        cardView.backgroundColor = .systemGray6
        cardView.layer.cornerRadius = 16
        cardView.layer.shadowColor = UIColor.black.cgColor
        cardView.layer.shadowOpacity = 0.2
        cardView.layer.shadowOffset = CGSize(width: 0, height: 4)
        cardView.translatesAutoresizingMaskIntoConstraints = false
        
        // 图片
        imageView.contentMode = .scaleAspectFill
        imageView.clipsToBounds = true
        imageView.layer.cornerRadius = 12
        imageView.backgroundColor = .systemBlue  // 占位色
        imageView.isUserInteractionEnabled = true  // 允许交互
        imageView.translatesAutoresizingMaskIntoConstraints = false
        
        // 标题
        titleLabel.text = "拖我试试"
        titleLabel.font = UIFont.systemFont(ofSize: 16, weight: .medium)
        titleLabel.textAlignment = .center
        titleLabel.translatesAutoresizingMaskIntoConstraints = false
        
        cardView.addSubview(imageView)
        cardView.addSubview(titleLabel)
        view.addSubview(cardView)
        
        // 布局
        NSLayoutConstraint.activate([
            cardView.centerXAnchor.constraint(equalTo: view.centerXAnchor),
            cardView.centerYAnchor.constraint(equalTo: view.centerYAnchor),
            cardView.widthAnchor.constraint(equalToConstant: 260),
            cardView.heightAnchor.constraint(equalToConstant: 300),
            
            imageView.topAnchor.constraint(equalTo: cardView.topAnchor, constant: 16),
            imageView.leadingAnchor.constraint(equalTo: cardView.leadingAnchor, constant: 16),
            imageView.trailingAnchor.constraint(equalTo: cardView.trailingAnchor, constant: -16),
            imageView.heightAnchor.constraint(equalToConstant: 180),
            
            titleLabel.topAnchor.constraint(equalTo: imageView.bottomAnchor, constant: 16),
            titleLabel.leadingAnchor.constraint(equalTo: cardView.leadingAnchor, constant: 16),
            titleLabel.trailingAnchor.constraint(equalTo: cardView.trailingAnchor, constant: -16)
        ])
        
        // 添加拖拽手势
        let panGesture = UIPanGestureRecognizer(target: self, 
                                                 action: #selector(handlePan(_:)))
        cardView.addGestureRecognizer(panGesture)
        
        // 添加点击手势（双击复位）
        let doubleTap = UITapGestureRecognizer(target: self, 
                                                action: #selector(handleDoubleTap(_:)))
        doubleTap.numberOfTapsRequired = 2
        cardView.addGestureRecognizer(doubleTap)
    }
    
    // 拖拽处理
    @objc func handlePan(_ gesture: UIPanGestureRecognizer) {
        switch gesture.state {
        case .began:
            cardOriginalCenter = cardView.center
        case .changed:
            let translation = gesture.translation(in: view)
            cardView.center = CGPoint(
                x: cardOriginalCenter.x + translation.x,
                y: cardOriginalCenter.y + translation.y
            )
        case .ended:
            // 如果拖出屏幕范围，弹回原位
            if cardView.frame.intersects(view.bounds) == false {
                UIView.animate(withDuration: 0.3) {
                    self.cardView.center = self.cardOriginalCenter
                }
            }
        default:
            break
        }
    }
    
    // 双击复位
    @objc func handleDoubleTap(_ gesture: UITapGestureRecognizer) {
        UIView.animate(withDuration: 0.3) {
            self.cardView.center = self.cardOriginalCenter
        }
    }
}
```

> 这个实战把前三篇的内容全部串起来了：UIView + Auto Layout（01.md）、viewDidLoad 一次性初始化（02.md）、Target-Action + 手势识别（本篇）。

## 关键要点总结

1. **触摸事件三阶段**：began → moved → ended/cancelled，本质是状态机
2. **命中测试（hitTest）**：从窗口向下递归，找到最深层被点中的视图——"谁被点了"
3. **响应链（Responder Chain）**：从被点中的视图向上冒泡，直到有人处理——"谁来处理"
4. **UIControl + Target-Action**：高层封装，将触摸翻译成语义化事件（.touchUpInside 等）
5. **UIGestureRecognizer**：手势识别器，处理点击/长按/拖拽/捏合等复杂手势
6. **UIImageView 默认不响应事件**：必须设置 `isUserInteractionEnabled = true`
