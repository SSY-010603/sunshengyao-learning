# 10 — 资源管理与国际化

> 本文覆盖公共概念：⑩ 资源管理与国际化

阶段 3 上一篇讲权限——用户隐私相关的上线要求。本篇讲**资源与展示相关的上线要求**：图片、字符串、颜色这些"静态资产"怎么组织、怎么适配多语言、怎么适配深色模式。这部分做不好，App 上线后会出现"某些文案没翻译"、"深色模式下图标看不清"、"不同设备字号错乱"这类体验问题，甚至被审核拒绝。

---

## 一句话定义

**资源管理与国际化** = 把 App 里用到的图片、字符串、颜色等静态资产从代码里剥离出来，按设备、语言、外观模式（浅色/深色）提供不同版本，让同一份代码适配全球用户和多种设备。

---

## C++ 直觉

```
C++ 里：
- 资源文件通常放在文件系统或打包进二进制（Windows 的 .rc、Qt 的 .qrc）
- 多语言用 gettext（.po/.mo 文件）或 Qt 的 tr() + .ts 文件
- 深色模式没有统一标准，各 GUI 框架自己实现

iOS 里：
- 资源放在 Asset Catalog（.xcassets），系统自动按设备/外观选择
- 国际化用 NSLocalizedString + .strings 文件
- 深色模式是 iOS 13+ 系统级支持，UIColor 自适应
```

## Asset Catalog — iOS 资源管理的大本营

### 为什么用 Asset Catalog

iOS 推荐用 **Asset Catalog（.xcassets）** 管理图片资源，而不是直接把 PNG/JPG 放在工程里。优势：

1. **自动切片**：系统帮你把 PDF 切成 1x/2x/3x（不同设备分辨率）
2. **自动适配**：深色模式、不同设备、不同语言自动切换
3. **按需加载**：App Thinning 按设备只打包对应资源，包体积小
4. **类型安全**：`[UIImage imageNamed:@"avatar"]` 直接通过名字取，编译期不报错但运行时如果名字错会得到 nil

### 图片资源的组织

在 Xcode 里 `Images.xcassets` 下创建 Image Set：

```
Images.xcassets/
├── AppIcon.appiconset/      ← App 图标（多尺寸）
├── LaunchImage.launchimage/ ← 启动图
├── avatar.imageset/         ← 普通图片
│   ├── Contents.json
│   ├── avatar@1x.png        ← 老设备
│   ├── avatar@2x.png        ← iPhone 8/SE
│   └── avatar@3x.png        ← iPhone 12/13/Pro
└── ...
```

代码里直接按名字取：

```objc
UIImage *img = [UIImage imageNamed:@"avatar"]; // 系统按设备自动选 @2x/@3x
```

### ⚠️ Asset Catalog 的坑

**坑 1：`imageNamed:` 会缓存**。图片加载后会驻留在内存，直到收到内存警告。大量图片场景（如 GIF 帧）应该用 `imageWithContentsOfFile:`：

```objc
// ✅ 大量图片用这个，不缓存
NSString *path = [[NSBundle mainBundle] pathForResource:@"frame_001" ofType:@"png"];
UIImage *img = [UIImage imageWithContentsOfFile:path];
```

**坑 2：图片名字拼错不报错**。`[UIImage imageNamed:@"avater"]`（拼错了）返回 nil，不会 crash 但界面会显示空白。开发时建议封装一个带断言的版本：

```objc
+ (UIImage *)safeImageNamed:(NSString *)name {
    UIImage *img = [UIImage imageNamed:name];
    NSAssert(img, @"图片资源不存在：%@", name);
    return img;
}
```

**坑 3：Bundle 和 Asset Catalog**。如果你的资源在子 Bundle（如 Framework 内的 .xcassets），要指定 Bundle：

```objc
UIImage *img = [UIImage imageNamed:@"avatar" inBundle:frameworkBundle compatibleWithTraitCollection:nil];
```

## 国际化（i18n）— 让 App 支持多语言

### 基本原理

iOS 用 **NSLocalizedString** + **.strings** 文件实现多语言：

```
1. 代码里用 NSLocalizedString 取字符串，不直接写死字面量
2. 每种语言一个 .strings 文件（如 en.lproj/Localizable.strings、zh-Hans.lproj/Localizable.strings）
3. App 运行时根据系统语言，自动加载对应的 .strings 文件
```

### 三步实现国际化

#### 第一步：代码里用 NSLocalizedString

```objc
// ❌ 写死中文
self.label.text = @"欢迎回来";

// ✅ 用 NSLocalizedString
self.label.text = NSLocalizedString(@"welcome_back", @"用户登录后的欢迎语");
// 第二个参数是注释，给翻译人员看上下文
```

#### 第二步：创建 .strings 文件

在 Xcode 中新建 Strings File（File → New → File → Strings File），命名为 `Localizable.strings`。然后在 File Inspector 里勾选支持的语言（如 English、Simplified Chinese）。

Xcode 会自动生成多语言版本：

```
Base.lproj/
└── Localizable.strings      ← 默认（通常英文）
en.lproj/
└── Localizable.strings      ← 英文
zh-Hans.lproj/
└── Localizable.strings      ← 简体中文
```

#### 第三步：填写翻译

```
// en.lproj/Localizable.strings
"welcome_back" = "Welcome back";
"like_count" = "%d likes";
"post_time" = "Posted %d minutes ago";

// zh-Hans.lproj/Localizable.strings
"welcome_back" = "欢迎回来";
"like_count" = "%d 人点赞";
"post_time" = "%d 分钟前发布";
```

代码里取：

```objc
self.label.text = NSLocalizedString(@"welcome_back", nil);

// 带参数的
NSInteger likes = 42;
NSString *likeText = [NSString stringWithFormat:NSLocalizedString(@"like_count", nil), likes];
// 英文： "42 likes"
// 中文： "42 人点赞"
```

### ⚠️ 国际化的坑

**坑 1：复数处理**。不同语言复数规则不同，英文有单复数（1 like / 2 likes），中文没有复数，阿拉伯语有单数/双数/复数三种。

iOS 13+ 用 `.stringsdict` 文件处理复数：

```xml
<!-- Localizable.stringsdict -->
<key>like_count</key>
<dict>
    <key>NSStringLocalizedFormatKey</key>
    <string>%#@likes@</string>
    <key>likes</key>
    <dict>
        <key>NSStringFormatSpecTypeKey</key>
        <string>NSStringPluralRuleType</string>
        <key>NSStringFormatValueTypeKey</key>
        <string>d</string>
        <key>one</key>
        <string>%d like</string>
        <key>other</key>
        <string>%d likes</string>
    </dict>
</dict>
```

```objc
// 英文环境
// likes=1 → "1 like"
// likes=5 → "5 likes"
NSString *text = [NSString localizedStringWithFormat:NSLocalizedString(@"like_count", nil), likes];
```

**坑 2：字符串拼接的陷阱**。不要拼接多个翻译片段：

```objc
// ❌ 拼接陷阱
NSString *text = [NSString stringWithFormat:@"%@ %@",
                    NSLocalizedString(@"you_have", nil),    // "你有"
                    NSLocalizedString(@"new_message", nil)]; // "新消息"
// 英文："You have new message"
// 中文："你有 新消息"（多一个空格，很丑）

// ✅ 用一个完整的翻译
NSString *text = [NSString stringWithFormat:NSLocalizedString(@"you_have_message", nil), count];
// 英文："You have %d new messages"
// 中文："你有 %d 条新消息"
```

不同语言**语序不同**。英文可能是"主语 + 谓语 + 宾语"，日文是"主语 + 宾语 + 谓语"。拼接片段会破坏语序。必须用带占位符的完整字符串。

**坑 3：日期和数字格式**。不同地区的日期/数字格式不同：

```objc
// ❌ 写死格式
NSString *dateStr = [NSString stringWithFormat:@"%04d-%02d-%02d", year, month, day];

// ✅ 用 NSDateFormatter，自动按地区格式
NSDateFormatter *formatter = [[NSDateFormatter alloc] init];
formatter.dateStyle = NSDateFormatterMediumStyle;
NSString *dateStr = [formatter stringFromDate:date];
// 美国： "Jul 23, 2026"
// 中国： "2026年7月23日"
// 德国： "23. Jul 2026"
```

**坑 4：文本长度变化**。翻译后字符串长度可能大幅变化：

```
"Save"       → 4 字符
"Speichern"  → 9 字符（德文）
"Сохранить"  → 8 字符（俄文）

"保存"        → 2 字符（中文）
```

设计 UI 时要考虑文本长度变化，避免固定宽度布局。Auto Layout + `numberOfLines = 0` + `adjustsFontSizeToFitWidth` 能缓解大多数情况。

## 深色模式（Dark Mode）

### iOS 13+ 系统级深色模式

iOS 13 引入深色模式，用户可在系统设置切换。App 要做的是让 UI 自适应。

### UIColor 自适应

#### 方式一：Asset Catalog 定义颜色

在 `Assets.xcassets` 里新建 Color Set，右侧面板可设浅色和深色两个版本：

```
BackgroundColor (Color Set)
├── Any Appearance ─── #FFFFFF (浅色)
└── Dark Appearance ─── #1C1C1E (深色)
```

代码里取：

```objc
// ⚠️ 要用 dynamicColorNamed: 而不是 colorWithName:
UIColor *bgColor = [UIColor dynamicColorNamed:@"BackgroundColor"];
self.view.backgroundColor = bgColor;
// 系统自动按当前外观选 #FFFFFF 或 #1C1C1E
```

#### 方式二：代码动态创建

```objc
// iOS 13+ 用 initWithDynamicProvider:
UIColor *dynamicColor = [UIColor colorWithDynamicProvider:^UIColor *(UITraitCollection *traitCollection) {
    if (traitCollection.userInterfaceStyle == UIUserInterfaceStyleDark) {
        return [UIColor colorWithRed:0.11 green:0.11 blue:0.12 alpha:1.0];
    } else {
        return [UIColor whiteColor];
    }
}];
```

### UIImage 自适应

深色模式下图标也要换。在 Asset Catalog 的 Image Set 里勾选 "Any Appearance" 和 "Dark"，分别提供两套图。

```objc
UIImage *logo = [UIImage imageNamed:@"logo"]; // 自动按外观选
```

### ⚠️ 深色模式的坑

**坑 1：硬编码颜色不自动切换**。

```objc
// ❌ 硬编码颜色，深色模式下白底白字看不见
self.label.textColor = [UIColor blackColor];

// ✅ 用系统动态颜色
self.label.textColor = [UIColor labelColor]; // iOS 13+ 系统提供
// 或从 Asset Catalog 取
```

**坑 2：视图重建后 trait 变化**。UITableViewCell 复用时颜色可能没更新，要监听 `traitCollectionDidChange:`：

```objc
- (void)traitCollectionDidChange:(UITraitCollection *)previousTraitCollection {
    [super traitCollectionDidChange:previousTraitCollection];
    if (previousTraitCollection.userInterfaceStyle != self.traitCollection.userInterfaceStyle) {
        // 外观切换了，更新颜色/图片
        [self updateColorsForCurrentTrait];
    }
}
```

**坑 3：调试技巧**。Xcode 模拟器 / 真机的 Debug Menu → `Environment Overrides` 可以临时切换外观，不用每次去系统设置改。

### 强制外观模式

有些 App 设计要求统一浅色或深色，在 Info.plist 里设：

```xml
<key>UIUserInterfaceStyle</key>
<string>Light</string>  <!-- 或 Dark -->
```

或在代码里：

```objc
// 在 AppDelegate 里
self.window.overrideUserInterfaceStyle = UIUserInterfaceStyleDark;
```

> 推荐支持两种外观切换，但保留一个"跟随系统"选项，用户体验更好。

## 字体适配

### Dynamic Type — 用户可调字号

iOS 允许用户在系统设置里调字号，App 要响应：

```objc
// ❌ 写死字号
label.font = [UIFont systemFontOfSize:14];

// ✅ 用 Dynamic Type
label.font = [UIFont preferredFontForTextStyle:UIFontTextStyleBody];
label.adjustsFontForContentSizeCategory = YES; // 用户调字号后自动更新
```

TextStyle 有几种：

| TextStyle | 默认字号 | 用途 |
|-----------|----------|------|
| UIFontTextStyleLargeTitle | 34 | 大标题 |
| UIFontTextStyleTitle1 | 28 | 一级标题 |
| UIFontTextStyleTitle2 | 22 | 二级标题 |
| UIFontTextStyleHeadline | 17 加粗 | 小标题 |
| UIFontTextStyleBody | 17 | 正文 |
| UIFontTextStyleCallout | 16 | 注释 |
| UIFontTextStyleFootnote | 13 | 脚注 |
| UIFontTextStyleCaption1 | 12 | 说明文字 |

用户在系统设置调大字号后，用 preferredFont 取的字体自动变大。**写死字号 `systemFontOfSize:14` 的字体不会变**。

### Auto Layout 配合字号

字号变大会挤乱布局，Auto Layout 配合 `numberOfLines = 0` 和 `numberOfLines = 0` 能让 Label 自动撑高：

```objc
label.numberOfLines = 0;  // 0 = 不限行数
[label setContentCompressionResistancePriority:UILayoutPriorityRequired forAxis:UILayoutAxisVertical];
```

## 资源管理的包体积优化

### Asset Catalog 的 App Thinning

App Store 上传后，Apple 会按设备生成专属包，只包含该设备需要的资源：

- iPhone 8（@2x）只下载 @2x 图片，不下载 @3x
- iPhone 12 Pro Max（@3x）只下载 @3x

你不用自己处理，Xcode + App Store 自动搞定。但你要确保**没在代码里硬引用某分辨率的图片**。

### 按需下载资源

大型资源（如游戏的关卡资源）可以放在 On-Demand Resources：

```objc
NSBundleResourceRequest *request = [[NSBundleResourceRequest alloc] initWithTags:@[@"level_1"]];
[request beginAccessingResourcesWithCompletionHandler:^(NSError *error) {
    if (!error) {
        // 资源下载完成，可以加载
        NSString *path = [request pathForResource:@"level_1_map" ofType:@"json"];
    }
}];
```

## 实战模式：统一资源管理

实际项目中，一般封装一个 Resource 工具类，集中管理字符串、颜色、图片的取用：

```objc
// R.swift 类似工具的简化版
@interface R : NSObject
+ (NSString *)string:(NSString *)key;
+ (UIColor *)color:(NSString *)name;
+ (UIImage *)image:(NSString *)name;
@end

@implementation R

+ (NSString *)string:(NSString *)key {
    NSString *value = NSLocalizedString(key, nil);
    NSAssert(value, @"缺少字符串资源：%@", key);
    return value;
}

+ (UIColor *)color:(NSString *)name {
    UIColor *color = [UIColor dynamicColorNamed:name];
    if (!color) color = [UIColor colorWithRed:0.5 green:0.5 blue:0.5 alpha:1.0];
    return color;
}

+ (UIImage *)image:(NSString *)name {
    UIImage *img = [UIImage imageNamed:name];
    NSAssert(img, @"缺少图片资源：%@", name);
    return img;
}

@end
```

业务里用：

```objc
self.label.text = [R string:@"welcome_back"];
self.view.backgroundColor = [R color:@"BackgroundColor"];
self.imageView.image = [R image:@"avatar"];
```

好处：集中管理、便于排查、后期换资源管理方案只改一处。
