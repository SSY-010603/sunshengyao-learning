# iOS 阶段 2 学习报告

## 学了什么

用四篇文章走完了 iOS 数据流转的全链路：

**05 · 状态管理与数据绑定** — 数据变了 UI 怎么跟着变，有三条路。KVO 是属性级观察者，运行时动态创建子类 hook setter，属性走 setter 修改时自动通知观察者，但必须 removeObserver 否则 crash，直接改 ivar 不触发。NSNotification 是全局广播，发收双方完全解耦，系统通知（前后台切换等）是最常见用例。Delegate + 手动刷新最朴素也最常用。上层架构用 MVVM，但 ObjC 的 MVVM 是"手动 MVVM"——没有框架自动绑定，核心规则是 VC 不碰 Model、ViewModel 不引用 UIKit。列表刷新的最大坑：先改 Model 再通知 View，否则 UITableView crash。

**06 · 网络通信** — URLSession 三件套（Configuration + Session + DataTask）是 iOS HTTP 客户端的基础。GET/POST 请求的回调在子线程，更新 UI 必须 dispatch 回主线程。JSON 解析用 NSJSONSerialization，映射成 Foundation 对象，但要注意 null 是 NSNull 不是 nil、数字和字符串类型可能混淆、数组元素类型不一定一致。实际业务用 YYModel 一行转模型。错误分三层：网络层（没网/超时）→ HTTP 层（4xx/5xx）→ 业务层（code 非 0）。图片加载不用自己写，SDWebImage 一行搞定下载+内存缓存+磁盘缓存。项目一般封装 NetworkManager 统一处理 token 和公共 header。ATS 要求必须 HTTPS。

**07 · 数据持久化** — 四层方案递增。UserDefaults 存少量键值，不是立即写盘、类型要匹配、不能存大数据。沙盒目录分 Documents（持久+备份）、Library/Caches（缓存+可清理）、tmp（临时）。NSCache 是内存级 LRU 缓存，系统内存不足自动淘汰。数据库用 FMDB 封装 SQLite，占位符防注入、FMDatabaseQueue 保线程安全、事务保原子性、PRAGMA user_version 做迁移。Core Data 了解即可，新项目不建议用。选方案看数据类型：偏好用 UserDefaults、缓存用 Caches 目录、用户主数据用数据库。

**08 · 多线程与异步编程** — 主线程铁律：所有 UI 操作必须在主线程。GCD 是系统级线程池调度器，队列分串行和并发，系统预置主队列和不同 QoS 的全局队列。dispatch_sync 对当前队列调用会死锁。五个常用模式：后台+回主线程（80% 场景）、dispatch_group 等多任务完成、dispatch_after 延迟、dispatch_once 单例、dispatch_barrier 读写锁。OperationQueue 支持取消、依赖和并发数控制。三个线程陷阱：多线程改可变集合 crash、循环中 Block 捕获变量时机、异步回调 self 生命周期（weak-strong dance）。

## 现在能做什么

独立完成带数据流转的完整功能，从接口请求 → JSON 解析 → 状态管理 → UI 展示 → 本地缓存 → 异步回主线程更新，全链路打通。具体包括：

- 写一个列表页：网络请求拿数据，YYModel 转模型，UITableView 展示，下拉刷新上拉加载
- 写一个详情页：接口取详情数据，展示到 UI，图片用 SDWebImage 加载
- 写一个设置页：UserDefaults 存偏好，NSNotificationCenter 监听全局事件（如登录/登出）
- 做本地缓存：接口数据写 FMDB，离线可读
- 控制并发：OperationQueue 限制图片下载并发数，dispatch_group 等多个接口都回来再刷新

常规业务需求（列表、详情、表单、登录注册、本地缓存）可以独立承接了。

## 还不能做什么

还不能独立交付上线级功能。上线需要关注的方面还没覆盖：运行时权限申请（相机、定位、相册）、资源管理与国际化（多语言、深色模式适配）、各种边界场景处理（弱网、空数据、错误兜底）。这些属于阶段 3。
