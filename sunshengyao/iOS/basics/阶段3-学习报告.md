# iOS 阶段 3 学习报告

## 学了什么

用两篇文章走完了 iOS 上线级能力的最后两块拼图：

**09 · 权限管理** — iOS 权限管理四步流程：Info.plist 声明用途 → 检查当前授权状态 → 请求授权触发系统弹窗 → 处理结果。Info.plist 声明是审核红线，没声明直接 crash，文案模糊（如"需要权限"）会被拒，必须写清具体用途（如"用于拍摄商品照片上传"）。所有权限都是四态状态机：未决定（NotDetermined）、已授权（Authorized）、已拒绝（Denied）、受限（Restricted）。相机的 API 是 `AVCaptureDevice requestAccessForMediaType:`，回调在任意线程需回主线程。相册权限 iOS 14+ 新增 Limited 模式（用户只授权部分照片），读写权限分离用不同的 Info.plist 键。定位权限有使用时（WhenInUse）和始终（Always）两级授权，后台定位审核极严必须说明充分理由。ATT 跟踪权限是 iOS 14.5+ 引入的，跨 App 跟踪用户前必须请求授权，是 IDFA 获取的前提。用户可随时在系统设置撤销权限，App 进入前台时要重新检查状态并提供降级方案。授权时机最忌在 App 启动时一次申请一堆，推荐用户主动触发时请求，配合"预解释"模式（自己先弹一个说明页再调系统弹窗）能让授权率提升 30-50%。

**10 · 资源管理与国际化** — iOS 资源管理用 Asset Catalog（.xcassets）集中管理图片，系统自动按设备分辨率选 1x/2x/3x、自动适配深色模式、App Thinning 按设备只打包对应资源减小包体积。但 imageNamed 会缓存图片在内存，大量图片场景应该用 imageWithContentsOfFile 避免缓存。国际化用 NSLocalizedString + .strings 文件，代码里用 key 取字符串、不同语言各有 .strings 文件、系统按用户语言自动加载。复数处理用 .stringsdict 配合 NSStringPluralRuleType。字符串拼接是国际化大坑——不同语言语序不同，不能把翻译片段拼起来，必须用带占位符的完整字符串。日期和数字格式化用 NSDateFormatter / NSNumberFormatter，自动按地区格式输出。深色模式 iOS 13+ 系统级支持，用 dynamicColorNamed 从 Asset Catalog 取颜色、Image Set 里同时提供浅色深色两套图、UITableViewCell 复用时监听 traitCollectionDidChange 更新颜色。字号适配用 Dynamic Type 的 preferredFontForTextStyle 配合 adjustsFontForContentSizeCategory。实战推荐封装一个 R 工具类统一管理字符串、颜色、图片，集中处理断言和兜底逻辑。

## 现在能做什么

独立交付符合上线标准的功能，覆盖了 iOS 开发的完整能力栈：

- **UI 开发**：拼 UI、布局、事件处理、页面跳转（阶段 1）
- **数据流转**：网络请求、JSON 解析、状态管理、本地缓存、异步编程（阶段 2）
- **上线能力**：权限申请、多语言适配、深色模式、资源管理、Dynamic Type 字号适配（阶段 3）
- **Crash 排障**：zombie（EXC_BAD_ACCESS）和 SIGABRT（NSException）两大类高频 crash 的定位、排障、生产监控（crash/01-zombie、02-sigabrt）
- **ObjC 语法基础**：类与对象、Foundation 数据类型、@property 与 ARC、Protocol/Category、Block 与 GCD（ObjC/01-05）

可以独立承接需要上线的完整需求，包括含权限、多语言、深色模式适配的功能；可以独立排查并修复线上 crash 问题；具备 iOS 端正式 owner 资格。

## 还有什么可以做

- **综合训练**：完整做一个小型 App（如简单的二手交易平台），把三个阶段的能力串起来，检验掌握程度
- **深入学习其他 crash 类型**：卡死/主线程无响应、内存溢出 OOM、KVO 异常深入等
- **学习跨端迁移**：基于 iOS 知识，用对照模式加速学习 Android 或鸿蒙
- **学习 Swift**：虽然 ObjC 是日常主力，但 Swift 是趋势，未来有需求时可以快速上手
