# iOS 阶段 1 学习报告

## 学了什么

用四篇文章走完了 iOS UIKit 的入门骨架：

**01 · UI 组件体系** — UIView 是一切界面的基类，视图之间有父子关系，形成一棵树。布局靠 Auto Layout 设约束，想省事就用 UIStackView 自动排列。UIKit 是命令式的（一步步告诉系统怎么做），SwiftUI 是声明式的（描述界面应该长什么样），先学 UIKit 打底。

**02 · 生命周期** — UIViewController 代表一个页面，从创建到销毁依次触发 viewDidLoad → viewWillAppear → viewDidAppear → viewWillDisappear → viewDidDisappear → deinit。核心区分：viewDidLoad 只调一次，viewWillAppear/viewDidAppear 每次可见都触发（因为导航栈不会销毁被盖住的页面）。最大的坑：忘了清理监听器和定时器会内存泄漏，iOS 没有 RAII 兜底。

**03 · 事件处理** — 用户触摸屏幕，系统先通过 hitTest 从上往下找"谁被点了"，再通过响应链从下往上找"谁来处理"。日常开发用不到底层 touches 回调，大部分时候用 UIControl 的 Target-Action（语义化事件如 .touchUpInside）和 UIGestureRecognizer（手势识别）就够了。别忘了 UIImageView 默认不响应事件，得手动开 isUserInteractionEnabled。

**04 · 导航与路由** — 页面跳转就是栈操作：Push 入栈、Pop 出栈。UINavigationController 管上下级跳转（右侧滑入），Present 管临时弹出（底部弹出）。传值三板斧：直接赋值正向传、闭包简单反向传、Delegate 复杂反向传。App 的典型结构是 TabBar 套 NavigationController——同级 Tab 切换，上下级 Push 跳转，临时流程 Present 弹出。

## 现在能做什么

根据设计稿搭出多页面的 App 骨架，包括：拼 UI、布局、处理按钮点击和手势、页面跳转和返回、页面之间传数据。纯 UI 改造类的需求可以独立接了。

## 还不能做什么

不能做带数据流转的完整功能——接口请求、数据展示、本地缓存这些都还没学。

## 下一步

阶段 2 涵盖四个概念：状态管理与数据绑定、网络通信、数据持久化、多线程与异步编程。学完后可以独立完成"从接口拿数据 → 展示到列表 → 点击进详情 → 缓存到本地"这类完整链路。
