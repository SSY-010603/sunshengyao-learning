# 06 — 网络通信

> 本文覆盖公共概念：⑥ 网络通信

阶段 1 你学会了拼 UI，阶段 2 上篇你学了数据变了怎么通知 UI 刷新。但数据从哪来？绝大部分来自服务端 API。这篇讲 iOS ObjC 里怎么做网络请求、怎么解析 JSON、怎么处理错误。

---

## 一句话定义

**网络通信** = 客户端向服务端发 HTTP 请求 → 拿到响应 → 解析数据 → 更新 UI。

## URLSession — iOS 的 HTTP 客户端

### 核心三件套

URLSession 由三个对象协作：

```
URLSessionConfiguration — 配置（超时、缓存策略、Cookie 等）
URLSession              — 会话（一个 configuration 创建一个 session）
URLSessionDataTask      — 具体的请求任务
```

类似 C++ 里 libcurl 的 `CURLM *multi = curl_multi_init()` 管理 `CURL *easy`，URLSession 是会话管理器，DataTask 是单个请求。

### 最简 GET 请求

```objc
- (void)fetchUserInfo {
    // 1. 构造 URL
    NSURL *url = [NSURL URLWithString:@"https://api.example.com/user/profile"];
    
    // 2. 创建 request（GET 是默认方法，不用显式设）
    NSURLRequest *request = [NSURLRequest requestWithURL:url];
    
    // 3. 创建 session（用 default 配置即可）
    NSURLSession *session = [NSURLSession sharedSession];
    
    // 4. 创建 dataTask，回调在子线程
    NSURLSessionDataTask *task = [session dataTaskWithRequest:request
            completionHandler:^(NSData *data, NSURLResponse *response, NSError *error) {
        if (error) {
            NSLog(@"请求失败：%@", error.localizedDescription);
            return;
        }
        
        // 5. 解析 JSON
        NSDictionary *json = [NSJSONSerialization JSONObjectWithData:data
                                                            options:0
                                                              error:nil];
        // 6. 回主线程更新 UI
        dispatch_async(dispatch_get_main_queue(), ^{
            self.nameLabel.text = json[@"name"];
        });
    }];
    
    // ⚠️ 必须手动 resume，task 创建后是挂起状态
    [task resume];
}
```

**关键点**：`dataTaskWithRequest:completionHandler:` 的回调**不在主线程**。更新 UI 必须 `dispatch_async(dispatch_get_main_queue(), ...)` —— 这和 C++ GUI 程序一个道理：不能在 IO 线程直接操作 UI 控件。

### POST 请求（提交 JSON Body）

```objc
- (void)loginWithPhone:(NSString *)phone password:(NSString *)password {
    NSURL *url = [NSURL URLWithString:@"https://api.example.com/user/login"];
    
    // NSMutableRequest 才能设 HTTP 方法和 body
    NSMutableURLRequest *request = [NSMutableURLRequest requestWithURL:url];
    request.HTTPMethod = @"POST";
    [request setValue:@"application/json" forHTTPHeaderField:@"Content-Type"];
    
    // 构造 body 字典 → JSON Data
    NSDictionary *body = @{@"phone": phone, @"password": password};
    request.HTTPBody = [NSJSONSerialization dataWithJSONObject:body
                                                       options:0
                                                         error:nil];
    
    [[NSURLSession sharedSession] dataTaskWithRequest:request
            completionHandler:^(NSData *data, NSURLResponse *response, NSError *error) {
        // 同上：检查 error → 解析 JSON → 回主线程刷 UI
    }] resume];
}
```

### 三种 Session Configuration

| 类型 | 说明 | 典型场景 |
|------|------|----------|
| `default` | 使用磁盘缓存、Cookie、Credential | 普通业务请求 |
| `ephemeral` | 不使用任何持久化存储（内存缓存） | 隐私模式、无痕浏览 |
| `background` | App 挂起/终止后仍可上传下载 | 大文件下载、后台传输 |

```objc
// 后台下载配置
NSURLSessionConfiguration *config = [NSURLSessionConfiguration backgroundSessionConfigurationWithIdentifier:@"com.app.download"];
NSURLSession *session = [NSURLSession sessionWithConfiguration:config delegate:self];
```

## JSON 解析 — NSJSONSerialization

### ObjC 的 JSON 处理比 C++ 简单

C++ 用 rapidjson 需要手动处理 DOM 节点类型、检查 `HasMember`、处理 `GetStringLength`，ObjC 的 NSJSONSerialization 直接把 JSON 映射成 Foundation 对象：

| JSON 类型 | Foundation 类型 |
|-----------|----------------|
| object `{}` | NSDictionary |
| array `[]` | NSArray |
| string | NSString |
| number | NSNumber |
| true/false | NSNumber (boolValue) |
| null | NSNull |

```objc
// 解析
NSData *jsonData = ...; // 从网络拿到的
NSDictionary *dict = [NSJSONSerialization JSONObjectWithData:jsonData options:0 error:nil];
NSString *name = dict[@"name"];          // 直接 subscript 取值
NSArray *list = dict[@"items"];          // 取数组
NSNumber *count = dict[@"total"];       // 取数值

// 生成
NSDictionary *body = @{@"action": @"like", @"id": @(12345)};
NSData *jsonBody = [NSJSONSerialization dataWithJSONObject:body options:0 error:nil];
```

### ⚠️ JSON 解析的坑

```objc
// 坑 1：服务端返回 null → ObjC 是 NSNull，不是 nil
// NSNull 调任何方法会 crash（NSNull 只有一个方法 null）
id value = dict[@"nickname"];
if (value && ![value isKindOfClass:[NSNull class]]) {
    self.nicknameLabel.text = value;
}

// 坑 2：服务端返回数字 → 你当字符串用，不 crash 但逻辑错
// 服务端 {"id": 12345} → dict[@"id"] 是 NSNumber
// 直接 stringWithFormat 没问题，但 length、substring 等字符串方法会 crash
NSNumber *userId = dict[@"id"];  // 不是 NSString！

// 坑 3：数组里的元素类型不一定一致
// 服务端可能返回 [{"type":1}, {"type":"hot"}]  → 第一个是 NSNumber，第二个是 NSString
for (NSDictionary *item in list) {
    id type = item[@"type"];
    if ([type isKindOfClass:[NSNumber class]]) {
        // ...
    }
}
```

### 手动解析 vs 模型转换（YYModel）

上面的 subscript 方式叫"手动解析"，简单但容易出错。实际业务中几乎都用 **YYModel**（或 MJExtension）自动把 JSON 字典转成 Model 对象：

```objc
// === Model 定义 ===
@interface FeedItem : NSObject
@property (nonatomic, copy) NSString *title;
@property (nonatomic, copy) NSString *coverURL;
@property (nonatomic, assign) NSInteger likeCount;
@property (nonatomic, assign) BOOL isLiked;
@end

// === 一行转模型 ===
FeedItem *item = [FeedItem yy_modelWithDictionary:dict];

// === 数组转模型 ===
NSArray<FeedItem *> *items = [FeedItem yy_modelArrayWithJson:listArray];
```

YYModel 的优势：
- 自动类型转换（`"123"` 字符串 → `NSInteger 123`）
- `null` 自动过滤，不会 crash
- 支持嵌套模型（`@property FeedAuthor *author`）
- 支持 key 映射（`+ (NSDictionary *)modelCustomPropertyMapper`）

> 你日常开发中大概率项目已经集成了 YYModel 或 MJExtension，直接用就行。理解底层 NSJSONSerialization 是为了出问题时能排查。

## 错误处理与重试

### NSError — ObjC 的错误对象

C++ 用异常或错误码，ObjC 用 NSError 指针：

```objc
// ObjC 错误传递模式：最后一个参数是 NSError **
NSError *error = nil;
NSDictionary *dict = [NSJSONSerialization JSONObjectWithData:data
                                                    options:0
                                                      error:&error];
if (error) {
    NSLog(@"JSON 解析失败：%@", error.localizedDescription);
}
```

### 网络请求的错误分类

```objc
[session dataTaskWithRequest:request
        completionHandler:^(NSData *data, NSURLResponse *response, NSError *error) {
    // 1. 网络层错误：没网、超时、DNS 解析失败
    if (error) {
        NSInteger code = error.code;
        if (code == NSURLErrorNotConnectedToInternet ||
            code == NSURLErrorTimedOut) {
            // 提示用户 "网络不给力"
            [self showNetworkError];
        }
        return;
    }
    
    // 2. HTTP 层错误：服务端返回了响应，但状态码不是 2xx
    NSHTTPURLResponse *httpResp = (NSHTTPURLResponse *)response;
    if (httpResp.statusCode >= 400) {
        // 解析服务端返回的错误信息
        NSDictionary *errorBody = [NSJSONSerialization JSONObjectWithData:data
                                                                 options:0
                                                                   error:nil];
        NSString *msg = errorBody[@"message"] ?: @"服务器开小差了";
        [self showError:msg];
        return;
    }
    
    // 3. 业务层错误：HTTP 200 但 code != 0
    NSDictionary *json = [NSJSONSerialization JSONObjectWithData:data options:0 error:nil];
    if ([json[@"code"] integerValue] != 0) {
        // 服务端定义的业务错误码
        [self showError:json[@"msg"]];
        return;
    }
    
    // 4. 成功
    NSDictionary *result = json[@"data"];
    // 更新 UI...
}]
```

### 简单重试策略

```objc
- (void)requestWithRetry:(NSInteger)retryCount {
    [session dataTaskWithRequest:request completionHandler:^(NSData *data, NSURLResponse *resp, NSError *error) {
        if (error && retryCount > 0) {
            // 网络错误，延迟 1 秒重试
            dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(1.0 * NSEC_PER_SEC)),
                           dispatch_get_main_queue(), ^{
                [self requestWithRetry:retryCount - 1];
            });
            return;
        }
        // 处理结果...
    }] resume];
}

// 调用：最多重试 2 次
[self requestWithRetry:2];
```

> 生产环境建议用 AFNetworking 或 Alamofire 的内置重试机制，不要自己写轮子。

## 图片加载 — SDWebImage

网络请求另一个高频场景是图片加载。日常开发中**不需要自己写图片下载**，直接用 SDWebImage：

```objc
// 一行加载网络图片 + 自动缓存
[self.imageView sd_setImageWithURL:[NSURL URLWithString:@"https://img.example.com/cover.jpg"]
                      placeholderImage:[UIImage imageNamed:@"placeholder"]];
```

SDWebImage 自动做了这些事：
1. 检查内存缓存 → 有则直接返回
2. 检查磁盘缓存 → 有则解压后返回
3. 都没有 → 发 HTTP 请求下载
4. 下载完成后写入磁盘缓存 + 内存缓存
5. UIImageView 退出屏幕后内存缓存自动释放（ARC）

```objc
// 常用扩展：加载失败时设置默认图
[self.imageView sd_setImageWithURL:url
                  placeholderImage:[UIImage imageNamed:@"default"]
                           completed:^(UIImage *image, NSError *error, SDImageCacheType cacheType, NSURL *url) {
    if (error) {
        self.imageView.image = [UIImage imageNamed:@"error_placeholder"];
    }
}];
```

## 实战封装 — 一个简单的 NetworkManager

实际项目中一般会封装一个统一的网络层，处理公共逻辑（token、签名、错误码、loading）：

```objc
@interface NetworkManager : NSObject
+ (instancetype)shared;
- (void)GET:(NSString *)path
     params:(NSDictionary *)params
    success:(void(^)(NSDictionary *data))success
    failure:(void(^)(NSString *msg))failure;
@end

@implementation NetworkManager

- (void)GET:(NSString *)path
     params:(NSDictionary *)params
    success:(void(^)(NSDictionary *data))success
    failure:(void(^)(NSString *msg))failure {
    
    // 1. 拼 URL + query string
    NSMutableString *urlStr = [NSMutableString stringWithFormat:@"%@%@", kBaseURL, path];
    if (params.count) {
        [urlStr appendString:@"?"];
        [params enumerateKeysAndObjectsUsingBlock:^(id key, id obj, BOOL *stop) {
            [urlStr appendFormat:@"%@=%@&", key, obj];
        }];
    }
    
    NSMutableURLRequest *request = [NSMutableURLRequest requestWithURL:[NSURL URLWithString:urlStr]];
    
    // 2. 加公共 header（token、版本号等）
    [request setValue:[self currentToken] forHTTPHeaderField:@"Authorization"];
    [request setValue:kAppVersion forHTTPHeaderField:@"X-App-Version"];
    
    // 3. 发请求
    [[NSURLSession sharedSession] dataTaskWithRequest:request
            completionHandler:^(NSData *data, NSURLResponse *response, NSError *error) {
        dispatch_async(dispatch_get_main_queue(), ^{
            if (error) {
                failure ? failure(@"网络不给力") : nil;
                return;
            }
            NSHTTPURLResponse *httpResp = (NSHTTPURLResponse *)response;
            if (httpResp.statusCode >= 400) {
                failure ? failure(@"服务器开小差了") : nil;
                return;
            }
            NSDictionary *json = [NSJSONSerialization JSONObjectWithData:data options:0 error:nil];
            if ([json[@"code"] integerValue] != 0) {
                failure ? failure(json[@"msg"]) : nil;
                return;
            }
            success ? success(json[@"data"]) : nil;
        });
    }] resume];
}

@end
```

业务 VC 里的用法就非常简洁：

```objc
[[NetworkManager shared] GET:@"/feed/list"
                      params:@{@"page": @(1)}
                     success:^(NSDictionary *data) {
    self.items = [FeedItem yy_modelArrayWithJson:data[@"list"]];
    [self.tableView reloadData];
} failure:^(NSString *msg) {
    [self showError:msg];
}];
```

## HTTPS 与 ATS

iOS 9+ 默认启用 **ATS (App Transport Security)**，要求所有 HTTP 请求走 HTTPS：

- 必须用 TLS 1.2+
- 证书必须是可信 CA 签发
- 不允许明文 HTTP

如果你的测试环境需要 HTTP，在 `Info.plist` 里临时豁免：

```xml
<!-- 允许指定域名走 HTTP -->
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSExceptionDomains</key>
    <dict>
        <key>test.api.example.com</key>
        <dict>
            <key>NSExceptionAllowsInsecureHTTPLoads</key>
            <true/>
        </dict>
    </dict>
</dict>
```

> **上线前必须去掉这个豁免**，Apple 审核 会检查 ATS 配置。

**06.md 总结**：

iOS 网络通信的核心是 URLSession 三件套：Configuration 配置会话参数、Session 是请求管理器、DataTask 是具体请求任务。发起请求后回调在子线程，更新 UI 必须 dispatch 回主线程。JSON 解析用 NSJSONSerialization，它把 JSON 直接映射成 Foundation 对象（NSDictionary/NSArray/NSString/NSNumber），但要注意三个坑：服务端返回 null 对应的是 NSNull 不是 nil、数字类型和字符串类型可能不一致、数组元素类型不一定统一。实际业务中基本都用 YYModel 做字典转模型，一行代码搞定类型转换和 null 过滤。错误处理分三层：网络层错误（没网、超时）→ HTTP 层错误（状态码 4xx/5xx）→ 业务层错误（HTTP 200 但 code 非 0）。图片加载不用自己写，SDWebImage 一行搞定下载+内存缓存+磁盘缓存。实际项目会封装一个 NetworkManager 统一处理 token、公共 header 和错误码。另外 iOS 的 ATS 机制要求必须走 HTTPS，测试环境需要在 Info.plist 临时豁免，上线前必须去掉。
