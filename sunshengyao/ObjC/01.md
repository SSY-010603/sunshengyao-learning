# ObjC 语法 #1：类与对象、消息发送、方法声明

> 你有 C++ 背景，ObjC 的面向对象思想和 C++ 一致，但语法长得完全不同。本文只讲差异和核心语法，不重复 OOP 基础。

---

## 1. 消息发送——ObjC 的灵魂

C++ 调方法：`obj->method(arg)`，编译时绑定。

ObjC 发消息：`[obj method:arg]`，运行时动态查找。

```objc
// C++: dog->bark();
// ObjC:
[dog bark];

// C++: dog->setName("Rex", age: 3);
// ObjC: 参数嵌入方法名
[dog setName:@"Rex" age:3];
//       ↑ 这是方法名的一部分，不是参数标签
//       方法名是 "setName:age:"  （注意冒号也是方法名的一部分）
```

**关键区别**：

| | C++ | ObjC |
|---|---|---|
| 调用方式 | `obj->method(arg)` | `[obj method:arg]` |
| 绑定时机 | 编译时（虚表） | 运行时（`objc_msgSend` 查 isa） |
| 方法名风格 | `setName(string, int)` | `setName:age:`（参数名是方法名的一部分） |
| 空指针调方法 | 💥 崩溃 | 给 nil 发消息返回 0/nil/NO，**不崩** |

> **给 nil 发消息不崩** 是 ObjC 最重要的安全特性。`[nil doSomething]` 返回 0（整数类型）、nil（对象类型）、NO（BOOL 类型）、0.0（浮点类型）。C++ 里对 nullptr 调成员函数是 UB。

## 2. 类的声明与实现

C++ 里类的声明和实现在同一个 `{}` 块里。ObjC 拆成两块：`@interface`（声明）和 `@implementation`（实现），通常分放在 `.h` 和 `.m` 文件。

### .h 文件（声明）

```objc
#import <UIKit/UIKit.h>          // 类似 C++ 的 #include，但不会重复导入

@interface Dog : NSObject        // 类名 : 父类（类似 class Dog : public NSObject）

// ---- 属性 ----
@property (nonatomic, copy) NSString *name;   // 后面 03.md 详解
@property (nonatomic, assign) NSInteger age;

// ---- 方法 ----
// 实例方法（- 号开头，类似 C++ 的普通成员函数）
- (void)bark;
- (NSString *)getName;
- (void)setName:(NSString *)name age:(NSInteger)age;

// 类方法（+ 号开头，类似 C++ 的 static 成员函数）
+ (instancetype)dogWithName:(NSString *)name;
+ (Dog *)defaultDog;

@end
```

### .m 文件（实现）

```objc
#import "Dog.h"

@implementation Dog

- (void)bark {
    NSLog(@"Woof! I'm %@", self.name);  // NSLog ≈ printf，@"" 是 ObjC 字符串字面量
}

- (NSString *)getName {
    return self.name;  // self ≈ C++ 的 this，但 self 是指针不需要 ->
}

- (void)setName:(NSString *)name age:(NSInteger)age {
    self.name = name;  // self.name 等价于 [self name] / [self setName:name]
    _age = age;        // _age 是属性的实例变量（ivar），直接访问不走 setter
}

+ (instancetype)dogWithName:(NSString *)name {
    Dog *dog = [[Dog alloc] init];  // 两步：alloc 分配内存 + init 初始化
    dog.name = name;
    return dog;
}

@end
```

## 3. 方法签名速查

```objc
// 无参数
- (返回类型)方法名;
- (void)bark;                    // void bark()

// 一个参数
- (返回类型)方法名:(参数类型)参数名;
- (void)setName:(NSString *)name;   // void setName(NSString* name)

// 多个参数——参数名是方法名的一部分
- (返回类型)方法名:(类型1)参数1 第二段:(类型2)参数2;
- (void)setName:(NSString *)name age:(NSInteger)age;
// 完整方法名 = "setName:age:"    // void setName_age(NSString* name, NSInteger age)

// 习惯写法（让方法名读起来像自然语言）
- (void)setRed:(CGFloat)red green:(CGFloat)green blue:(CGFloat)blue;
// 调用: [color setRed:1.0 green:0.5 blue:0.0]
// 完整方法名 = "setRed:green:blue:"
```

**vs C++ 对比**：

```cpp
// C++
class Dog {
public:
    void bark();
    void setName(string name, int age);
    static Dog* create(string name);
};

// ObjC 等价
@interface Dog : NSObject
- (void)bark;
- (void)setName:(NSString *)name age:(NSInteger)age;
+ (instancetype)dogWithName:(NSString *)name;
@end
```

## 4. 对象创建与初始化

```objc
// C++: Dog d;  或  Dog* d = new Dog("Rex");
// ObjC: 必须在堆上，没有栈对象

// 方式 1：标准两步
Dog *dog = [[Dog alloc] init];
//          ↑ alloc: 分配内存（返回未初始化的对象）
//                 ↑ init: 初始化（返回初始化后的对象，可能返回 nil）

// 方式 2：工厂方法（如果有定义）
Dog *dog = [Dog dogWithName:@"Rex"];

// 方式 3：new（只是 alloc+init 的简写，不推荐——不能自定义 init 参数）
Dog *dog = [Dog new];  // 等价于 [[Dog alloc] init]
```

### 自定义 init

```objc
// .h
- (instancetype)initWithName:(NSString *)name age:(NSInteger)age;

// .m
- (instancetype)initWithName:(NSString *)name age:(NSInteger)age {
    self = [super init];         // 1. 调父类 init
    if (self) {                  // 2. 检查是否成功（super init 可能返回 nil）
        _name = [name copy];     // 3. 初始化自己的属性
        _age = age;
    }
    return self;                 // 4. 返回 self
}
```

> `instancetype` vs `id`：`instancetype` 是返回类型，告诉编译器返回的是该类的实例（编译器能做类型检查）。`id` 是万能指针类型（类似 C++ 的 `void*`），编译器不做类型检查。init 系列方法用 `instancetype`，其他不确定类型时用 `id`。

## 5. 头文件管理

```objc
// #import ≈ #include + #pragma once（自动防止重复导入）
#import <UIKit/UIKit.h>      // 系统框架：尖括号
#import "Dog.h"              // 自己的头文件：双引号

// @class 前向声明（类似 C++ 的 class Dog;）
@class Cat;                  // .h 里只声明，不导入 Cat.h，减少编译依赖
@interface Dog : NSObject
@property (nonatomic, weak) Cat *friend;  // 用 @class 声明后就能用指针
@end
// .m 里再 #import "Cat.h" 获取完整接口
```

## 6. self 和 super

```objc
// self ≈ C++ 的 this（但 self 是关键字，不是隐式参数）
self.name = @"Rex";     // 等价于 [self setName:@"Rex"]（走 setter）
_name = @"Rex";          // 直接访问实例变量，不走 setter

// super 不是指针，是编译器指令——告诉 runtime 跳过当前类的方法分派，从父类开始查找
- (void)viewDidLoad {
    [super viewDidLoad];  // 调用 UIViewController 的 viewDidLoad
}
```

> **self.name vs _name**：`self.name` 走 setter（触发 KVO、copy 语义等），`_name` 直接访问实例变量。通常在 init 和 dealloc 里用 `_name`（避免触发子类重写的 setter），其他地方用 `self.name`。

## 7. 常用惯用语法速查

```objc
// 字符串：@"" 前缀
NSString *s = @"Hello";           // C++: std::string s = "Hello";

// 数字：@() 语法
NSNumber *n = @42;                // 等价于 [NSNumber numberWithInt:42]
NSNumber *d = @3.14;              // [NSNumber numberWithDouble:3.14]
NSNumber *b = @YES;               // [NSNumber numberWithBool:YES]

// 数组：@[] 语法
NSArray *arr = @[@"a", @"b"];     // 等价于 [NSArray arrayWithObjects:@"a", @"b", nil]
// ⚠️ nil 是数组结束标记，不能放 nil 进数组

// 字典：@{} 语法
NSDictionary *dict = @{@"key": @"value"};  // [NSDictionary dictionaryWithObject:@"value" forKey:@"key"]

// 可变版本
NSMutableArray *mArr = [NSMutableArray array];
[mArr addObject:@"new"];          // 添加元素
NSMutableDictionary *mDict = [NSMutableDictionary dictionary];
[mDict setObject:@"val" forKey:@"k"];
```

## 关键要点

1. **消息发送 `[obj method:arg]`** 不是函数调用，是运行时动态分派（`objc_msgSend`）
2. **给 nil 发消息不崩**，返回 0/nil/NO——这是 ObjC 最重要的安全特性
3. **方法名包含冒号**：`setName:age:` 是一个完整方法名，不是两个
4. **`@interface` / `@implementation`** 分开放 .h/.m，类似 C++ 声明与实现分离
5. **`instancetype`** 用于 init 返回类型，**`id`** 是万能类型（类似 `void*`）
6. **`self.name` 走 setter，`_name` 直接访问 ivar**——init/dealloc 里用 `_name`
