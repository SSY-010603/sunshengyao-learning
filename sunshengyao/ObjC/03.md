# ObjC 语法 #3：@property 与 ARC 内存管理

> 这是你日常写 ObjC 最容易出错的地方。@property 的修饰词组合决定了内存安全，ARC 帮你自动管理引用计数但不包打一切。

---

## 1. @property 做了什么

一行 `@property` 实际生成了三样东西：

```objc
@property (nonatomic, copy) NSString *name;
// 自动生成：
// 1. 实例变量 _name
// 2. getter: - (NSString *)name;
// 3. setter: - (void)setName:(NSString *)name;
```

你可以重写 getter/setter（但重写后就不会自动生成实例变量，除非用 `@synthesize`）。

### 访问方式

```objc
self.name = @"Rex";       // 走 setter（触发 copy 语义、KVO 通知等）
_name = @"Rex";            // 直接赋值实例变量，跳过 setter
[self setName:@"Rex"];     // 等价于 self.name = @"Rex"
NSString *n = self.name;   // 走 getter
NSString *n2 = _name;      // 直接读 ivar
NSString *n3 = [self name]; // 等价于 self.name
```

> **何时用 self.name vs _name**：
> - `init` 里用 `_name`（避免触发子类重写的 setter）
> - `dealloc` 里用 `_name`（同理）
> - 其他地方用 `self.name`（保证语义一致性 + KVO 兼容）

## 2. @property 修饰词全解

`@property (修饰词1, 修饰词2, ...) 类型 名字;`

### 2.1 线程安全：nonatomic vs atomic

| 修饰词 | 行为 | 选择 |
|--------|------|------|
| `nonatomic` | 不加锁，直接读写 | **99% 场景用这个** |
| `atomic` | 加锁保证原子性 | 几乎不用（锁粒度太大，且不保证线程安全） |

```objc
@property (nonatomic, copy) NSString *name;    // ✅ 推荐
@property (atomic, copy) NSString *name;       // ❌ 不推荐（性能差，且不能替代真正的线程安全）
```

> `atomic` 只保证 getter/setter 单次操作的原子性，不保证"读-改-写"的线程安全。真正的线程安全需要用锁或 GCD。

### 2.2 内存管理：strong / weak / copy / assign / unsafe_unretained

| 修饰词 | 引用计数 | 对象释放后 | 适用类型 | 使用场景 |
|--------|---------|-----------|---------|---------|
| `strong` | +1 | 不影响（引用存在期间不释放） | 对象类型 | **默认**，持有对象 |
| `weak` | 0 | **自动置 nil** | 对象类型 | delegate、outlet、避免循环引用 |
| `copy` | +1（拷贝） | 不影响 | NSString / block | **NSString 必须用 copy**，block 也用 copy |
| `assign` | 0 | **不置 nil → zombie** | 非对象类型 | NSInteger / CGFloat / CGPoint 等 |
| `unsafe_unretained` | 0 | **不置 nil → zombie** | 对象类型 | 极罕见，ARC 之前遗留 |

```objc
// ✅ 常见正确组合
@property (nonatomic, copy)   NSString *title;         // NSString 用 copy！
@property (nonatomic, strong) UIView *contentView;      // UI 控件用 strong
@property (nonatomic, weak)   id<SomeDelegate> delegate; // delegate 用 weak
@property (nonatomic, assign) NSInteger count;          // 标量用 assign
@property (nonatomic, copy)   void (^completionBlock)(void); // block 用 copy
```

### ⚠️ 为什么 NSString 必须用 copy

```objc
// ❌ 用 strong —— 可能被外部修改
@property (nonatomic, strong) NSString *title;

NSMutableString *mutableStr = [NSMutableString stringWithString:@"Hello"];
obj.title = mutableStr;      // strong 只保留引用，指向同一个 NSMutableString
[mutableStr appendString:@" World"];
// obj.title 变成了 "Hello World" —— 你被外部改了！

// ✅ 用 copy —— 拷贝一份不可变副本
@property (nonatomic, copy) NSString *title;

obj.title = mutableStr;       // copy 语义：setter 里执行 [mutableStr copy]
                               // 拷贝出不可变的 NSString，和原 NSMutableString 脱钩
[mutableStr appendString:@" World"];
// obj.title 还是 "Hello" —— 安全
```

**copy 的 setter 实际做的事**：

```objc
- (void)setTitle:(NSString *)title {
    _title = [title copy];    // 如果传入 NSMutableString，copy 返回不可变 NSString
                               // 如果传入 NSString，copy 返回自身（浅拷贝，无性能损耗）
}
```

> **C++ 类比**：`copy` ≈ `std::string` 的值语义（拷贝独立），`strong` ≈ `std::shared_ptr`（共享所有权）。

## 3. ARC 内存管理核心

### 3.1 ARC 做什么

ARC 在编译时自动在代码里插入 `retain`/`release`/`autorelease` 调用：

```objc
// 你写的代码
- (void)doSomething {
    Dog *dog = [[Dog alloc] init];  // 引用计数 = 1
    [dog bark];
}   // 离开作用域，ARC 自动插入 [dog release]，引用计数 = 0，dealloc

// 编译器实际生成的（简化）
- (void)doSomething {
    Dog *dog = [[Dog alloc] init];
    [dog retain];       // ARC 插入
    [dog bark];
    [dog release];      // ARC 插入
}
```

你**不需要也不允许**手动写 `retain`/`release`/`autorelease`/`dealloc`（释放对象的逻辑）。

### 3.2 引用计数规则

```
alloc / new / copy / mutableCopy  → 引用计数 = 1（你拥有它）
retain                            → 引用计数 +1
release                           → 引用计数 -1
引用计数 = 0                       → dealloc 被调用，对象销毁
```

**ARC 下你只需要关心：用 strong 还是 weak 还是 copy？** 计数加减编译器全包了。

### 3.3 weak 的生命周期

```objc
__weak typeof(self) weakSelf = self;
// weakSelf 不增加引用计数
// 当 self 被 dealloc 后，runtime 自动将 weakSelf 置为 nil
// 再次访问 weakSelf → nil，不崩

// 内部实现：runtime 维护一个 weak 引用表（哈希表）
// 对象 dealloc 时，遍历所有指向它的 weak 引用，逐个置 nil
```

### 3.4 autorelease 与 autoreleasepool

```objc
// autorelease：延迟释放，当前 runloop 结束时 release
- (Dog *)createDog {
    Dog *dog = [[Dog alloc] init];  // 引用计数 = 1
    return dog;                      // ARC 自动插入 [dog autorelease]
    // 调用方在当前 runloop 周期结束前持有 dog，之后 autorelease pool drain → release
}

// 手动 autoreleasepool（大量临时对象时用，防止内存峰值）
for (int i = 0; i < 100000; i++) {
    @autoreleasepool {
        NSString *str = [NSString stringWithFormat:@"%d", i];  // 临时对象
        // str 在这个 autoreleasepool 块结束时就被释放，不用等到 runloop 结束
    }
}
```

## 4. 循环引用与破环

### 4.1 典型循环引用

```objc
// ❌ 循环引用：self → block → self
@interface MyVC : UIViewController
@property (nonatomic, copy) void (^onComplete)(void);
@end

- (void)setup {
    self.onComplete = ^{
        [self doSomething];   // block 强引用 self，self 强引用 block
    };
}
// self → onComplete(block) → self → 永远不会 dealloc
```

### 4.2 破环：weak-strong dance

```objc
- (void)setup {
    __weak typeof(self) weakSelf = self;     // weak 引用，不增加引用计数
    self.onComplete = ^{
        __strong typeof(weakSelf) strongSelf = weakSelf;  // block 执行期间强持有
        if (!strongSelf) return;             // self 已释放，安全退出
        [strongSelf doSomething];           // 执行期间 self 不会被释放
    };
}
```

**为什么要 strongSelf？** 只有 weak 不用 strong 的话，block 执行期间 self 可能被其他线程释放，导致 block 执行到一半 self 变 zombie。`__strong` 保证 block 执行期间 self 存活。

### 4.3 delegate 循环引用

```objc
// ❌ strong delegate → 循环引用
@property (nonatomic, strong) id<SomeDelegate> delegate;

// ✅ weak delegate → 不构成循环
@property (nonatomic, weak) id<SomeDelegate> delegate;
```

### 4.4 NSTimer 循环引用

```objc
// ❌ target-action → timer 强引用 target → 循环引用
self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0
                                               target:self
                                             selector:@selector(tick)
                                             userInfo:nil
                                              repeats:YES];

// ✅ iOS 10+ block API + weak self
__weak typeof(self) weakSelf = self;
self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0
                                             repeats:YES
                                               block:^(NSTimer *t) {
    [weakSelf tick];
}];
```

## 5. dealloc 注意事项

ARC 下你不需要 `[super dealloc]`（编译器自动插入），但你仍然需要写 `dealloc` 来清理非 ARC 管理的资源：

```objc
- (void)dealloc {
    // ✅ 需要手动清理的：
    [self.timer invalidate];              // NSTimer 必须手动 invalidate
    [[NSNotificationCenter defaultCenter] removeObserver:self];  // 通知
    // KVO 的 removeObserver
    
    // ❌ 不需要做的：
    // [super dealloc];                   // ARC 自动插入
    // self.someStrongProperty = nil;      // ARC 会自动 release
}
```

## 关键要点

1. **@property 一行生成三样东西**：ivar + getter + setter
2. **NSString 必须用 copy**——防止外部 NSMutableString 修改；其他对象类型默认 strong
3. **delegate 必须用 weak**——防止循环引用
4. **标量用 assign**（NSInteger / CGFloat / struct），对象类型不能用 assign
5. **ARC 自动管理引用计数**，你只需选对 strong/weak/copy
6. **weak-strong dance** 是破环的标准写法：weak 打破循环，strong 保证执行安全
7. **dealloc 里只清理非 ARC 资源**（timer、通知、KVO），不要手动 release
