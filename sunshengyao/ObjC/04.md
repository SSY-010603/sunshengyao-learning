# ObjC 语法 #4：Protocol 协议与 Category 分类

> Protocol ≈ C++ 纯虚类（接口），Category ≈ 给已有类"追加方法"（C++ 没有对应机制）。

---

## 1. Protocol（协议）

### 1.1 声明协议

```objc
// 类似 C++ 的纯虚类 / Java 的 interface
@protocol SomeDelegate <NSObject>       // <NSObject> 表示继承 NSObject 协议

// 必须实现的方法（@required 是默认）
@required
- (void)didFinishLoading:(NSDictionary *)result;
- (BOOL)shouldRetry;

// 可选实现的方法
@optional
- (void)didProgressUpdate:(CGFloat)progress;
- (NSString *)customTitle;

@end
```

### 1.2 遵循协议

```objc
// .h — 声明遵循
@interface MyViewController : UIViewController <SomeDelegate, AnotherDelegate>
//                         父类                <协议1, 协议2>
@end

// .m — 实现协议方法
@implementation MyViewController

- (void)didFinishLoading:(NSDictionary *)result {
    // 必须实现（@required）
}

- (BOOL)shouldRetry {
    return YES;
}

// @optional 的方法可以选择性实现
- (void)didProgressUpdate:(CGFloat)progress {
    // 可选
}

@end
```

### 1.3 用协议类型声明属性

```objc
@property (nonatomic, weak) id<SomeDelegate> delegate;
//                           ↑ id 类型 + 遵循 SomeDelegate 协议
// 编译器会检查：赋值给 delegate 的对象必须遵循 SomeDelegate

// 多个协议
@property (nonatomic, weak) id<SomeDelegate, AnotherDelegate> delegate;
```

### 1.4 检查可选方法是否实现

```objc
// ❌ 直接调用 @optional 方法——如果对方没实现会崩
[self.delegate didProgressUpdate:0.5];

// ✅ 先检查是否实现
if ([self.delegate respondsToSelector:@selector(didProgressUpdate:)]) {
    [self.delegate didProgressUpdate:0.5];
}
```

> **C++ 类比**：Protocol ≈ 纯虚基类（`class IDelegate { virtual void didFinish() = 0; }`），但 ObjC 的 `@optional` 让你可以不实现所有方法——C++ 纯虚函数必须全实现，ObjC 的 optional 让接口更灵活。

### 1.5 协议 vs 继承

| | 继承（: 父类） | 协议（<Protocol>） |
|---|---|---|
| 可以有几个 | 只能 1 个父类 | 可以遵循多个协议 |
| 有实现代码吗 | 父类有实现 | 协议**没有实现**（只有声明） |
| 类比 | C++ 单继承 | C++ 纯虚类 / Java interface |

## 2. Category（分类/类别）

Category 可以**给已有类追加方法**，不需要修改原始类的代码，甚至不需要源码。C++ 没有这个机制。

### 2.1 声明 Category

```objc
// 文件名惯例：ClassName+CategoryName.h / .m
// NSString+MyAdditions.h

#import <Foundation/Foundation.h>

@interface NSString (MyAdditions)     // (括号里是分类名)

- (NSString *)reverseString;          // 追加实例方法
+ (NSString *)emptyString;            // 追加类方法

@end
```

```objc
// NSString+MyAdditions.m

#import "NSString+MyAdditions.h"

@implementation NSString (MyAdditions)

- (NSString *)reverseString {
    NSUInteger len = [self length];
    NSMutableString *result = [NSMutableString stringWithCapacity:len];
    [self enumerateSubstringsInRange:NSMakeRange(0, len)
                               options:(NSStringEnumerationReverse | NSStringEnumerationByComposedCharacterSequences)
                            usingBlock:^(NSString *substring, NSRange substringRange, NSRange enclosingRange, BOOL *stop) {
        [result appendString:substring];
    }];
    return result;
}

+ (NSString *)emptyString {
    return @"";
}

@end
```

### 2.2 使用 Category

```objc
#import "NSString+MyAdditions.h"

NSString *s = @"Hello";
NSString *r = [s reverseString];       // → @"olleH" —— 好像 NSString 自带的方法
NSString *e = [NSString emptyString];   // → @""
```

### 2.3 Category 的限制

| 能做 | 不能做 |
|------|--------|
| 追加方法 | **追加实例变量**（不能加 property，除非是 `@property (nonatomic, assign)` 这种不生成 ivar 的） |
| 覆盖已有方法（**不推荐**） | 保证覆盖一定生效（多个 Category 覆盖同一方法时，最后一个编译的生效，行为不确定） |

```objc
// ❌ Category 里不能加 ivar
@interface Dog (Extra)
@property (nonatomic, strong) NSString *nickname;  // 编译报错：不能合成 ivar
@end

// ✅ 可以用关联对象模拟（见下文 2.5）
```

### 2.4 常见用途

```objc
// 1. 给系统类加便利方法
@interface UIColor (MyApp)
+ (UIColor *)myAppPrimaryColor;
+ (UIColor *)myAppBackgroundColor;
@end

// 2. 拆分大类的实现（把 VC 的不同逻辑拆到不同 Category）
// MyVC+Network.m   — 网络相关方法
// MyVC+UI.m         — UI 配置方法
// MyVC+DataSource.m — 数据源方法

// 3. 给类加私有方法（替代独立 .m 文件里的私有方法）
```

### 2.5 关联对象（Associated Object）——模拟添加 ivar

Category 不能加 ivar，但可以用 `objc_setAssociatedObject` 给对象动态挂一个值：

```objc
// .h
#import <objc/runtime.h>

@interface UIView (MyTag)
@property (nonatomic, copy) NSString *myTag;   // 看起来像 property，实际用关联对象实现
@end

// .m
@implementation UIView (MyTag)

static const void *kMyTagKey = &kMyTagKey;     // 关联对象的 key

- (NSString *)myTag {
    return objc_getAssociatedObject(self, kMyTagKey);
}

- (void)setMyTag:(NSString *)myTag {
    objc_setAssociatedObject(self, kMyTagKey, myTag, OBJC_ASSOCIATION_COPY_NONATOMIC);
    //                                         ↑ 内存策略：copy / retain / assign
}

@end
```

## 3. Extension（类扩展）

Extension 是一种**匿名的 Category**，可以在 .m 文件里声明"私有"属性和方法：

```objc
// MyVC.m — 在 .m 文件顶部
@interface MyVC ()    // 括号里是空的 = Extension

// 私有属性（不会暴露到 .h）
@property (nonatomic, strong) NSString *internalState;
@property (nonatomic, weak) id<SomeDelegate> delegate;

// 私有方法声明（前向声明，让编译器知道有这个方法）
- (void)privateHelper;

@end
```

**Extension vs Category**：

| | Extension | Category |
|---|---|---|
| 括号里 | `()` 空的 | `(CategoryName)` |
| 能加 ivar 吗 | ✅ 能（因为编译时和主类 @implementation 一起编译） | ❌ 不能 |
| 实现位置 | 必须在主类 `@implementation` 里 | 独立的 `@implementation ClassName (CategoryName)` |
| 用途 | 声明私有属性/方法 | 给类追加公开方法 |

> **日常写法**：`.h` 放公开接口，`.m` 顶部用 Extension 放私有属性和方法。这是 ObjC 最常见的文件组织方式。

## 4. id 和 instancetype 与协议组合

```objc
// id — 万能对象指针（类似 void*，编译器不做类型检查）
id obj = @"Hello";          // 任何 ObjC 对象
[obj length];                // 编译器不警告，即使 id 实际是 NSString
                             // 但如果 obj 不是 NSString，运行时崩溃

// id<Protocol> — 遵循协议的万能指针
id<SomeDelegate> delegate;   // 可以是任何遵循 SomeDelegate 的对象
[delegate didFinishLoading:result];  // 编译器知道有这个方法 ✅

// instancetype — 返回当前类的实例
// 用于 init 和工厂方法，编译器能推断具体类型
- (instancetype)init;                   // ✅
+ (instancetype)dogWithName:(NSString *)name;  // ✅

// ⚠️ 不要用 id 作为 init 返回类型
- (id)init;   // ❌ 编译器丢失类型信息
```

## 5. SEL 和 @selector

```objc
// SEL — 方法选择器（类型安全的方法指针名）
SEL sel = @selector(viewDidLoad);      // 获取方法选择器
SEL sel2 = NSSelectorFromString(@"viewDidLoad");  // 从字符串获取

// 检查对象是否实现了某个方法
if ([obj respondsToSelector:@selector(updateUI)]) {
    [obj performSelector:@selector(updateUI)];  // 动态调用
}

// performSelector ⚠️
// 编译器无法确定 performSelector: 调用的方法是否有返回值/参数
// ARC 下可能内存泄漏（编译器不知道方法名，无法插入正确的 release）
// 优先用 block / NSInvocation 替代

// 从 SEL 得到字符串
NSString *selName = NSStringFromSelector(@selector(viewDidLoad));  // → "viewDidLoad"
```

## 关键要点

1. **Protocol ≈ C++ 纯虚类**：声明接口不写实现，`@required` 必须实现，`@optional` 可选
2. **调用 @optional 方法前必须 `respondsToSelector:`**——否则没实现就崩
3. **Category 给已有类追加方法**——C++ 没有对应机制，这是 ObjC 的运行时动态特性
4. **Category 不能加 ivar**——需要用关联对象（Associated Object）模拟
5. **Extension = 匿名 Category**——可以加 ivar，用于 `.m` 里声明私有属性和方法
6. **`id<Protocol>` 是常用的类型声明**——比 `id` 多了协议约束
7. **`@selector` 获取方法名**——配合 `respondsToSelector:` / `performSelector:` 做动态分派
