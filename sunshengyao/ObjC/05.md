# ObjC 语法 #5：Block 闭包与 GCD

> Block ≈ C++ lambda，GCD 是 C 层面的并发 API。日常开发中两个一起用：Block 传回调，GCD 切线程。

---

## 1. Block 语法

### 1.1 声明与定义

```objc
// Block 声明语法：返回类型 (^变量名)(参数类型)
void (^simpleBlock)(void) = ^{
    NSLog(@"Hello from block");
};
simpleBlock();

// 带参数和返回值
int (^addBlock)(int, int) = ^(int a, int b) {
    return a + b;
};
int result = addBlock(3, 5);  // → 8
```

**vs C++ lambda**：

```cpp
// C++ lambda
auto add = [](int a, int b) -> int { return a + b; };

// ObjC Block
int (^add)(int, int) = ^(int a, int b) { return a + b; };
```

| | C++ lambda | ObjC Block |
|---|---|---|
| 声明 | `auto f = [](...){...};` | `返回类型 (^名字)(参数) = ^(...){...};` |
| 捕获 | `[=]` / `[&]` / `[x, &y]` | 自动捕获（值拷贝），`__block` 可修改 |
| 类型 | `std::function<Ret(Args...)>` | Block 指针类型 |
| 是对象吗 | 不是（除非包装进 std::function） | **是**（继承 NSObject，可 retain/release/copy） |

### 1.2 用 typedef 简化 Block 类型

```objc
// Block 类型太长，用 typedef 简化
typedef void (^CompletionBlock)(NSDictionary *result, NSError *error);
typedef void (^ProgressBlock)(CGFloat progress);

// 使用
@property (nonatomic, copy) CompletionBlock onComplete;
@property (nonatomic, copy) ProgressBlock onProgress;

- (void)loadDataWithCompletion:(CompletionBlock)completion
                      progress:(ProgressBlock)progress;
```

> **⚠️ Block 属性必须用 copy**：Block 初始在栈上（会被函数返回销毁），copy 会把 Block 拷贝到堆上（生命周期随对象）。ARC 下通常编译器自动 copy，但显式写 `copy` 是好习惯。

### 1.3 Block 作方法参数

```objc
// 声明
- (void)fetchDataWithCompletion:(void (^)(NSDictionary *result, NSError *error))completion;

// 调用
[obj fetchDataWithCompletion:^(NSDictionary *result, NSError *error) {
    if (error) {
        NSLog(@"Error: %@", error);
    } else {
        NSLog(@"Result: %@", result);
    }
}];
```

## 2. Block 捕获变量

### 2.1 默认值捕获（const 拷贝）

```objc
int value = 10;
void (^block)(void) = ^{
    NSLog(@"%d", value);   // ✅ 可以读取
    // value = 20;         // ❌ 编译报错：Block 里不能修改捕获的自动变量
};
value = 20;                 // Block 外修改不影响 Block 内——Block 捕获的是值拷贝
block();                    // 打印 10，不是 20
```

### 2.2 __block 可修改

```objc
__block int counter = 0;
void (^increment)(void) = ^{
    counter++;               // ✅ __block 变量可以在 Block 里修改
    NSLog(@"%d", counter);
};
increment();  // 打印 1
increment();  // 打印 2
NSLog(@"%d", counter);  // 打印 2 —— Block 外也能看到修改
```

> **C++ 类比**：ObjC 的默认捕获 ≈ C++ 的 `[=]`（值拷贝），`__block` ≈ `[&]`（引用捕获）。但 `__block` 的实现不是引用，是把变量包装在一个结构体里放堆上，Block 和外部都通过这个结构体访问。

### 2.3 捕获 self

```objc
// ❌ 直接捕获 self —— 强引用
void (^block)(void) = ^{
    [self doSomething];      // Block 强引用 self
};

// ✅ weak 引用 + strong 守卫
__weak typeof(self) weakSelf = self;
void (^block)(void) = ^{
    __strong typeof(weakSelf) strongSelf = weakSelf;
    if (!strongSelf) return;
    [strongSelf doSomething];
};
```

### 2.4 捕获 ObjC 对象

```objc
NSString *name = @"Rex";
void (^block)(void) = ^{
    NSLog(@"%@", name);      // ✅ Block 会 retain name（ARC 自动）
};
// Block 执行时 name 一定还活着——Block 强引用了它

NSMutableString *mutable = [NSMutableString stringWithString:@"Hello"];
void (^block2)(void) = ^{
    NSLog(@"%@", mutable);   // Block retain 了 mutable
    [mutable appendString:@" World"];  // ✅ 可以调用方法修改内容
};
```

## 3. Block 的内存管理

| Block 位置 | 何时 | 生命周期 |
|-----------|------|---------|
| **栈上** | 定义在函数/方法内，没被 copy | 函数返回即销毁 |
| **堆上** | 被 copy 后 | 随持有者生命周期 |
| **全局** | 不捕获任何自动变量 | 程序生命周期 |

```objc
// ⚠️ 经典陷阱：返回栈上的 Block
typedef void (^MyBlock)(void);

- (MyBlock)makeBlock {
    int value = 42;
    // ❌ 危险：Block 在栈上，函数返回后可能被销毁
    return ^{ NSLog(@"%d", value); };
    // ARC 下通常安全（编译器自动 copy 到堆）
    // MRC 下必崩
}
```

> **ARC 下编译器会自动 copy 的场景**：Block 赋值给 strong 属性、Block 作为函数返回值、Block 传给 Cocoa API。但传给 C 函数指针参数时不会自动 copy。

## 4. GCD（Grand Central Dispatch）

GCD 是 C 层面的并发 API，不依赖 ObjC runtime，但日常在 ObjC 里用得最多。

### 4.1 队列类型

```objc
// 串行队列 — 任务按顺序执行
dispatch_queue_t serialQueue = dispatch_queue_create("com.myapp.serial", DISPATCH_QUEUE_SERIAL);

// 并发队列 — 任务并行执行
dispatch_queue_t concurrentQueue = dispatch_queue_create("com.myapp.concurrent", DISPATCH_QUEUE_CONCURRENT);

// 系统全局并发队列（最常用，不用自己创建）
dispatch_queue_t globalQueue = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0);

// 主队列 — 串行，在主线程执行（UI 操作必须在这个队列）
dispatch_queue_t mainQueue = dispatch_get_main_queue();
```

| 队列 | 类型 | 线程 | 用途 |
|------|------|------|------|
| `mainQueue` | 串行 | 主线程 | **UI 更新必须在这** |
| `globalQueue` | 并发 | 子线程 | 网络请求、计算、IO |
| 自定义串行 | 串行 | 子线程 | 需要顺序执行的异步任务 |
| 自定义并发 | 并发 | 子线程 | 可并行的后台任务 |

### 4.2 异步派发（最常用）

```objc
// 异步派发到后台执行，不阻塞当前线程
dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
    // 后台线程：网络请求 / 耗时计算
    NSDictionary *result = [self fetchFromServer];
    
    // 回主线程更新 UI
    dispatch_async(dispatch_get_main_queue(), ^{
        self.label.text = result[@"title"];
    });
});

// ⚠️ 这个模式你每天都会写：后台干活 → 主线程更新 UI
```

### 4.3 同步派发（少用，容易死锁）

```objc
// 同步派发：等任务执行完才继续
dispatch_sync(serialQueue, ^{
    // 在 serialQueue 上执行
});
// 这里任务一定执行完了

// ❌ 经典死锁：在主队列上 sync 派发到主队列
dispatch_sync(dispatch_get_main_queue(), ^{
    // 主线程等主队列执行 → 主队列等主线程空闲 → 死锁 💥
});
```

> **规则**：永远不要在当前串行队列上 `dispatch_sync` 到自身。

### 4.4 dispatch_after — 延迟执行

```objc
// 延迟 2 秒在主线程执行
dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(2.0 * NSEC_PER_SEC)),
               dispatch_get_main_queue(), ^{
    NSLog(@"2 seconds later");
});
```

### 4.5 dispatch_group — 等待多个任务完成

```objc
dispatch_group_t group = dispatch_group_create();

// 任务 1
dispatch_group_async(group, dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
    [self fetchUserInfo];
});

// 任务 2
dispatch_group_async(group, dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
    [self fetchFeedList];
});

// 所有任务完成后回调
dispatch_group_notify(group, dispatch_get_main_queue(), ^{
    // 两个请求都完成了，更新 UI
    [self.tableView reloadData];
});
```

### 4.6 dispatch_once — 线程安全的一次性初始化

```objc
// 单例模式
+ (instancetype)sharedInstance {
    static MyClass *instance = nil;
    static dispatch_once_t onceToken;
    dispatch_once(&onceToken, ^{
        instance = [[MyClass alloc] init];
    });
    return instance;
}
// dispatch_once 保证即使多线程同时调用，初始化代码也只执行一次
// C++ 类比：std::call_once + std::once_flag
```

### 4.7 dispatch_barrier — 读写锁模式

```objc
// 并发队列上实现"多读单写"
dispatch_queue_t queue = dispatch_queue_create("com.myapp.rw", DISPATCH_QUEUE_CONCURRENT);

// 读操作 — 并发执行
dispatch_async(queue, ^{
    [self readData];
});

// 写操作 — barrier 确保独占
dispatch_barrier_async(queue, ^{
    [self writeData];
});
// barrier 之前的读操作全部完成后，才开始写操作
// 写操作完成后，barrier 之后的操作才继续并发执行
```

## 5. 常见 Block 使用模式

### 5.1 回调模式

```objc
// 网络请求回调（你日常写得最多的）
- (void)loadWithCompletion:(void (^)(id result, NSError *error))completion {
    dispatch_async(dispatch_get_global_queue(0, 0), ^{
        NSError *error = nil;
        id result = [self doNetworkRequestWithError:&error];
        dispatch_async(dispatch_get_main_queue(), ^{
            completion(result, error);
        });
    });
}

// 调用
[loader loadWithCompletion:^(id result, NSError *error) {
    if (error) { ... }
    else { self.data = result; }
}];
```

### 5.2 枚举回调

```objc
NSArray *arr = @[@"a", @"b", @"c"];
[arr enumerateObjectsUsingBlock:^(NSString *obj, NSUInteger idx, BOOL *stop) {
    NSLog(@"%lu: %@", (unsigned long)idx, obj);
    if ([obj isEqualToString:@"b"]) {
        *stop = YES;  // 提前终止
    }
}];
```

### 5.3 动画回调

```objc
[UIView animateWithDuration:0.3 animations:^{
    self.view.alpha = 0;
} completion:^(BOOL finished) {
    [self.view removeFromSuperview];
}];
```

## 关键要点

1. **Block ≈ C++ lambda**，但 Block 是对象（继承 NSObject），可以被 retain/release/copy
2. **Block 默认值捕获（const 拷贝）**，`__block` 才能修改——≈ C++ 的 `[=]` vs `[&]`
3. **Block 属性必须用 copy**——Block 初始在栈上，copy 拷贝到堆上
4. **weak-strong dance 破循环引用**：`__weak` 打破循环，`__strong` 保证执行安全
5. **GCD 三板斧**：`dispatch_async`(异步) + `dispatch_get_main_queue()`(主线程) + `dispatch_group`(等全部完成)
6. **后台干活 → 主线程更新 UI**，这是你每天写的模式
7. **`dispatch_sync` 别往当前串行队列派**——死锁
8. **`dispatch_once` 实现单例**——≈ C++ 的 `std::call_once`
