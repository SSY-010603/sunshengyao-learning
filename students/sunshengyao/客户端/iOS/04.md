## 核心概念：导航栈

iOS 的页面跳转基于**栈（Stack）**——和 C++ 中的 `std::stack` 是同一个概念：

- **Push**：新页面入栈，覆盖在当前页面上方
- **Pop**：最上面的页面出栈，回到前一个页面
- **栈底**：App 的根页面，不能被 pop 掉

```
导航栈（UINavigationController 管理）

┌──────────────────┐
│  详情页 (top)      │  ← 当前屏幕显示的
├──────────────────┤
│  列表页           │  ← 被盖住了，但还在栈里
├──────────────────┤
│  首页 (root)      │  ← 栈底，App 入口
└──────────────────┘
```

> **C++ 直觉**：这就是后进先出（LIFO）的栈。`push` = 压栈，`pop` = 弹栈。你在数据结构课上学过的，一模一样。

## 两种跳转方式：Push vs Present

iOS 有两种基本的页面跳转方式，效果完全不同：

### 1. Push（推入导航栈）

由 **UINavigationController** 管理。新页面从右侧滑入，左上角自动出现"返回"按钮。

```swift
// Push 跳转
let detailVC = DetailViewController()
navigationController?.pushViewController(detailVC, animated: true)

// Push 返回
navigationController?.popViewController(animated: true)
```

**视觉效果**：新页面从右侧滑入，返回时向右滑出。这是 iOS 最常见的跳转方式——微信的聊天列表→聊天详情、设置页→子设置页，都是 Push。

### 2. Present（模态弹出）

新页面从底部弹出，覆盖当前页面。没有自动的返回按钮，需要自己添加关闭逻辑。

```swift
// Present 弹出
let loginVC = LoginViewController()
loginVC.modalPresentationStyle = .fullScreen  // 全屏弹出（iOS 13+ 默认是卡片样式）
present(loginVC, animated: true)

// Dismiss 关闭
dismiss(animated: true)
```

**视觉效果**：新页面从底部向上弹出。常见场景：登录页、分享弹窗、相册选择器等"临时性"页面。

### 什么时候用 Push，什么时候用 Present？

| 方式 | 适用场景 | 返回方式 | 有导航栏吗 |
|------|---------|---------|-----------|
| **Push** | 层级式导航（列表→详情→子详情） | 左上角自动返回按钮 | ✅ 有 |
| **Present** | 临时性/独立流程（登录、分享、编辑） | 需手动 dismiss | ❌ 没有（可自己加） |

> **简单判断**：如果两个页面属于"同一流程的上下级"，用 Push；如果新页面是"另一个独立任务"，用 Present。

## UINavigationController 详解

UINavigationController 是导航栈的管理者。它自身不显示内容，而是管理一组 UIViewController 的入栈和出栈。

### 结构

```
UINavigationController
  ├── UINavigationBar     ← 顶部导航栏（系统自动管理）
  └── UIViewController    ← 当前显示的页面
       ├── view (内容区域)
       └── navigationItem ← 导航栏的配置（标题、按钮）
```

### 初始化

```swift
// 方法一：在 AppDelegate 中设置根导航控制器
let rootVC = HomeViewController()
let navigationController = UINavigationController(rootViewController: rootVC)
window?.rootViewController = navigationController

// 方法二：在 SceneDelegate 中（iOS 13+）
let rootVC = HomeViewController()
let navigationController = UINavigationController(rootViewController: rootVC)
window?.rootViewController = navigationController
```

### 导航栏配置

每个页面可以独立配置自己的导航栏外观：

```swift
override func viewDidLoad() {
    super.viewDidLoad()
    
    // 标题
    title = "首页"
    // 或用 navigationItem 更细粒度控制
    navigationItem.title = "首页"
    
    // 右侧按钮
    navigationItem.rightBarButtonItem = UIBarButtonItem(
        barButtonSystemItem: .add,
        target: self,
        action: #selector(addTapped)
    )
    
    // 左侧按钮（会替换默认的返回按钮）
    navigationItem.leftBarButtonItem = UIBarButtonItem(
        title: "关闭",
        style: .plain,
        target: self,
        action: #selector(closeTapped)
    )
    
    // 大标题样式（iOS 11+，类似微信/设置的标题）
    navigationItem.largeTitleDisplayMode = .always
    navigationController?.navigationBar.prefersLargeTitles = true
}
```

### 栈操作

```swift
// Push：推入新页面
navigationController?.pushViewController(detailVC, animated: true)

// Pop：返回上一页
navigationController?.popViewController(animated: true)

// Pop 到根页面（一次性返回多层）
navigationController?.popToRootViewController(animated: true)

// Pop 到指定页面
navigationController?.popToViewController(someVC, animated: true)

// 查看栈中所有页面
let allVCs = navigationController?.viewControllers  // [VC数组]
```

> **C++ 类比**：UINavigationController 就像你手写的栈管理器——`push`/`pop`/`top`/`size` 都有，只不过它管理的是 UIViewController 对象而不是 int。

## 页面间参数传递

页面跳转时传数据是日常开发的高频操作。有三种常用方式：

### 方式一：直接赋值（最常用）

在 Push/Present 之前，直接给目标页面的属性赋值：

```swift
// 发送方
let detailVC = DetailViewController()
detailVC.itemId = "12345"        // 直接赋值
detailVC.itemTitle = "商品名称"
navigationController?.pushViewController(detailVC, animated: true)

// 接收方（DetailViewController）
class DetailViewController: UIViewController {
    var itemId: String = ""       // 声明属性
    var itemTitle: String = ""
    
    override func viewDidLoad() {
        super.viewDidLoad()
        title = itemTitle         // 使用传入的数据
        loadDetail(id: itemId)
    }
}
```

> **C++ 类比**：这就是构造对象后设置成员变量，和 `obj.setId("12345")` 一样自然。

### 方式二：闭包回调（反向传值）

当目标页面需要把数据**回传**给来源页面时（比如选择了一个联系人后返回）：

```swift
// 发送方
class ContactListVC: UIViewController {
    var onContactSelected: ((String) -> Void)?  // 闭包属性
    
    func showContactPicker() {
        let pickerVC = ContactPickerVC()
        pickerVC.onSelect = { [weak self] contactId in
            self?.updateContact(contactId)
            self?.dismiss(animated: true)
        }
        present(pickerVC, animated: true)
    }
}

// 接收方（选择完成后回调）
class ContactPickerVC: UIViewController {
    var onSelect: ((String) -> Void)?
    
    func contactSelected(id: String) {
        onSelect?(id)  // 通过闭包回传数据
    }
}
```

> **C++ 类比**：类似 C++ 的 std::function 回调——你注册一个回调函数，目标页面在合适时机调用它。

### 方式三：Delegate 代理模式

这是 iOS 最经典的设计模式，适合复杂的反向通信：

```swift
// 1. 定义协议（类似 C++ 的纯虚类/接口）
protocol EditProfileDelegate: AnyObject {
    func editProfileDidFinish(name: String)
}

// 2. 接收方持有代理引用
class EditProfileVC: UIViewController {
    weak var delegate: EditProfileDelegate?
    
    func saveButtonTapped() {
        delegate?.editProfileDidFinish(name: newName)
        dismiss(animated: true)
    }
}

// 3. 发送方实现协议
class ProfileVC: UIViewController, EditProfileDelegate {
    func showEdit() {
        let editVC = EditProfileVC()
        editVC.delegate = self  // 设置代理
        present(editVC, animated: true)
    }
    
    func editProfileDidFinish(name: String) {
        // 收到回传数据
        updateNameLabel(name)
    }
}
```

> **C++ 类比**：Delegate 本质就是接口（纯虚类）。Swift 的 protocol ≈ C++ 的 abstract class with pure virtual functions。发送方通过接口指针调用方法，不需要知道具体实现。

### 三种方式对比

| 方式 | 适用方向 | 复杂度 | 典型场景 |
|------|---------|--------|---------|
| 直接赋值 | 正向（A→B） | 低 | 列表→详情，传 ID |
| 闭包回调 | 反向（B→A） | 中 | 选择器返回结果 |
| Delegate 代理 | 反向（B→A） | 高 | 复杂的多步回传 |

**日常选择**：正向传值用直接赋值，简单反向用闭包，复杂反向用 Delegate。

## UITabBarController：底部标签切换

除了导航栈的"上下级"跳转，App 通常还有"同级切换"——底部的 Tab 按钮：

```
UITabBarController
  ├── Tab 1: 首页 (UINavigationController → HomeVC)
  ├── Tab 2: 发现 (UINavigationController → DiscoverVC)  
  ├── Tab 3: 消息 (UINavigationController → MessageVC)
  └── Tab 4: 我的 (UINavigationController → ProfileVC)
```

### 初始化

```swift
let homeNav = UINavigationController(rootViewController: HomeVC())
homeNav.tabBarItem = UITabBarItem(title: "首页", image: UIImage(systemName: "house"), tag: 0)

let discoverNav = UINavigationController(rootViewController: DiscoverVC())
discoverNav.tabBarItem = UITabBarItem(title: "发现", image: UIImage(systemName: "magnifyingglass"), tag: 1)

let messageNav = UINavigationController(rootViewController: MessageVC())
messageNav.tabBarItem = UITabBarItem(title: "消息", image: UIImage(systemName: "message"), tag: 2)

let profileNav = UINavigationController(rootViewController: ProfileVC())
profileNav.tabBarItem = UITabBarItem(title: "我的", image: UIImage(systemName: "person"), tag: 3)

let tabBar = UITabBarController()
tabBar.viewControllers = [homeNav, discoverNav, messageNav, profileNav]

window?.rootViewController = tabBar
```

> **重要**：每个 Tab 内部通常是独立的 UINavigationController。这意味着不同 Tab 的导航栈是**隔离的**——首页的 Push 栈和消息页的 Push 栈互不干扰。

### 代码切换 Tab

```swift
// 切换到第 2 个 Tab
tabBarController?.selectedIndex = 1

// 获取当前选中的 Tab
let currentIndex = tabBarController?.selectedIndex
```

## 典型 App 架构：TabBar + NavigationController

大多数 iOS App 的根架构是：**UITabBarController 套 UINavigationController**：

```
UITabBarController (根)
  ├── UINavigationController (Tab 1)
  │    ├── HomeVC (root)
  │    ├── DetailVC (pushed)
  │    └── SubDetailVC (pushed)
  ├── UINavigationController (Tab 2)
  │    ├── DiscoverVC (root)
  │    └── SearchResultVC (pushed)
  ├── UINavigationController (Tab 3)
  │    └── MessageVC (root)
  └── UINavigationController (Tab 4)
       └── ProfileVC (root)
```

- **Tab 切换**：同级页面切换，用 `UITabBarController`
- **Push 跳转**：上下级页面，用 `UINavigationController`
- **Present 弹出**：临时流程，脱离导航栈

## 关键要点总结

1. **导航栈**：iOS 页面跳转基于栈，Push 入栈、Pop 出栈，和 C++ `std::stack` 概念一致
2. **两种跳转**：Push（层级导航，右侧滑入）vs Present（模态弹出，底部弹出）
3. **UINavigationController**：管理导航栈和导航栏，每个页面通过 `navigationItem` 配置标题和按钮
4. **三种传值方式**：直接赋值（正向）、闭包回调（简单反向）、Delegate 代理（复杂反向）
5. **UITabBarController**：底部标签切换，每个 Tab 内是独立的 UINavigationController
6. **典型架构**：TabBar + NavigationController，同级用 Tab 切换，上下级用 Push
