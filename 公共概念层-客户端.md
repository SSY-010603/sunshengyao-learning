# 公共概念层

> 本文档定义 Android / iOS / 鸿蒙 三端开发中共通的核心概念。
> 各端教学图谱引用这些概念，在跨端学习时自动标注"你已在 XX 端学过"。

---

## 概念 1：UI 组件体系

**定义**：用可嵌套的树状结构来组织和渲染用户界面的基本范式。

**覆盖内容**：
- 组件树 / View 层级 / Widget Tree
- 基础容器组件与叶子组件（如按钮、文本、图片）
- 组件属性与样式（颜色、大小、间距、字体）
- 自定义组件的创建方式
- 声明式 UI vs 命令式 UI 的两种范式差异

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| 组件基类 | View / ViewGroup | UIView | Component |
| 布局容器 | LinearLayout / ConstraintLayout | UIStackView / Auto Layout | Row / Column / Flex |
| 文本 | TextView | UILabel | Text |
| 按钮 | Button | UIButton | Button |
| 图片 | ImageView | UIImageView | Image |
| 列表 | RecyclerView | UITableView / UICollectionView | List |
| 声明式框架 | Jetpack Compose | SwiftUI | ArkUI |

## 概念 2：生命周期

**定义**：页面或组件从创建、可见、不可见到销毁的完整状态流转序列。理解生命周期是正确处理资源申请和释放的基础。

**覆盖内容**：
- 页面级生命周期（进入页面、离开页面、销毁）
- 组件级生命周期（创建、挂载、更新、卸载）
- 前后台切换（App 进入后台 vs 回到前台）
- 生命周期回调中的常见操作（初始化、保存状态、释放资源）

**跨端对应**：

| 阶段 | Android (Activity) | iOS (UIViewController) | 鸿蒙 (UIAbility / 组件) |
|------|---------------------|------------------------|--------------------------|
| 创建 | onCreate | viewDidLoad | aboutToAppear |
| 可见 | onStart / onResume | viewWillAppear / viewDidAppear | onPageShow |
| 不可见 | onPause / onStop | viewWillDisappear / viewDidDisappear | onPageHide |
| 销毁 | onDestroy | deinit / viewDidUnload | aboutToDisappear |
| 后台 | onStop | applicationDidEnterBackground | onBackground |
| 前台 | onRestart | applicationWillEnterForeground | onForeground |

## 概念 3：状态管理与数据绑定

**定义**：UI 如何响应数据变化 —— 当数据发生变化时，界面自动更新；当用户操作界面时，数据随之更新。

**覆盖内容**：
- 单向数据绑定 vs 双向数据绑定
- 观察者模式 / 发布订阅模式
- 状态提升 / 状态管理容器
- 不可变数据与响应式数据
- MVVM / MVI / MVP 等架构模式简介（概念层面）

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| 单向绑定 | DataBinding / Compose State | SwiftUI @State / @Binding | ArkUI @State / @Prop |
| 双向绑定 | Two-way DataBinding | Combine @Published | ArkUI @Link |
| 状态容器 | ViewModel + LiveData / StateFlow | ObservableObject / @StateObject | AppStorage / ViewModel |
| 架构模式 | MVVM（ViewModel） | MVVM / MV | MVVM |

---

## 概念 4：事件处理与用户交互

**定义**：用户对屏幕的操作（点击、滑动、手势）如何被系统捕获并转化为代码可以响应的事件。

**覆盖内容**：
- 事件分发机制（事件如何从系统传递到目标组件）
- 点击、长按、拖拽等基础事件
- 手势识别（捏合、旋转、滑动）
- 事件冒泡与拦截
- 触摸事件的坐标和状态（down / move / up / cancel）

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| 点击 | setOnClickListener | addTarget / .onTapGesture | onClick |
| 手势 | GestureDetector | UIGestureRecognizer | gesture() |
| 事件分发 | dispatchTouchEvent / onInterceptTouchEvent | hitTest / responder chain | 事件响应链 |
| 列表项点击 | Adapter + onClick | didSelectRowAt | onItemClick |

---

## 概念 5：导航与路由

**定义**：页面之间的跳转、参数传递、返回操作的管理机制。

**覆盖内容**：
- 页面栈（导航栈）的概念
- 页面跳转（push / present）
- 页面返回（pop / dismiss）
- 页面间参数传递
- 深层链接（Deep Link）/ 路由 Scheme
- Tab 切换 / 底部导航

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| 跳转 | Intent + startActivity | pushViewController / NavigationLink | router.pushUrl |
| 返回 | finish() / onBackPressed | popViewController | router.back |
| 参数传递 | Intent extras / Bundle | prepare(for:sender:) / init | router params |
| 底部Tab | BottomNavigationView | UITabBarController | Tabs |
| 声明式导航 | NavHost + NavController | NavigationStack | Navigation |

## 概念 6：网络通信

**定义**：客户端如何与服务器交换数据 —— 包括请求发送、响应解析、错误处理。

**覆盖内容**：
- HTTP 请求方法（GET / POST / PUT / DELETE）
- 请求与响应的结构（Header、Body、状态码）
- JSON 序列化与反序列化
- 异步请求与回调 / async-await
- 错误处理与重试策略
- 图片加载与缓存（概念层面）

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| HTTP 客户端 | OkHttp / Retrofit | URLSession / Alamofire | @ohos.net.http |
| JSON 解析 | Gson / Moshi / kotlinx.serialization | Codable / JSONDecoder | JSON.parse |
| 图片加载 | Glide / Coil | SDWebImage / Kingfisher | Image 组件 |
| 异步处理 | Coroutines / RxJava | async/await / Combine | async/await |

---

## 概念 7：数据持久化

**定义**：数据如何在应用关闭后依然保存，并在下次启动时恢复。

**覆盖内容**：
- 键值对存储（简单配置 / 偏好设置）
- 结构化数据库（SQLite / ORM）
- 文件存储（沙盒目录结构）
- 缓存策略（内存缓存 vs 磁盘缓存）
- 数据迁移与版本管理

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| 键值存储 | SharedPreferences / DataStore | UserDefaults | Preferences |
| ORM / 数据库 | Room (SQLite) | Core Data / GRDB (SQLite) | RelationalStore |
| 文件目录 | getFilesDir() / getCacheDir() | Documents / Caches / tmp | 应用沙箱目录 |
| 数据库升级 | Room Migration | Core Data Migration | 数据库版本升级 |

---

## 概念 8：多线程与异步编程

**定义**：如何在后台执行耗时任务，同时保持 UI 响应流畅；多任务并发的管理方式。

**覆盖内容**：
- 主线程（UI 线程）的概念和限制
- 将耗时任务放到后台线程执行
- 线程间通信 —— 后台任务完成后回到主线程更新 UI
- 串行 vs 并发任务
- 协程 / async-await / GCD 等异步编程模型
- 常见陷阱：内存泄漏、死锁、线程安全

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| 主线程 | Main Thread / UI Thread | Main Thread / Main Queue | 主线程 |
| 后台任务 | Coroutines / WorkManager | GCD (DispatchQueue) / OperationQueue | TaskPool / Worker |
| 回主线程 | withContext(Dispatchers.Main) | DispatchQueue.main.async | postTask |
| 取消任务 | Job.cancel() | Operation.cancel() / Task.cancel() | task.cancel() |

---

## 概念 9：权限管理

**定义**：应用访问敏感数据或硬件（相机、位置、通讯录等）时，需要向系统和用户申请授权。

**覆盖内容**：
- 权限分类（普通权限 vs 危险权限 / 运行时权限）
- 权限申请流程（声明 → 检查 → 请求 → 处理结果）
- 用户拒绝后的处理策略
- 隐私权限（相册、通讯录、定位、相机、麦克风）

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| 权限声明 | AndroidManifest.xml | Info.plist | module.json5 |
| 运行时请求 | ActivityCompat.requestPermissions | requestAuthorization | requestPermissionsFromUser |
| 回调处理 | onRequestPermissionsResult | completion handler | PermissionRequestResult |
| 权限组 | 危险权限分为多个组 | 每项权限单独请求 | 按权限类型分组 |

---

## 概念 10：资源管理与国际化

**定义**：应用如何使用和管理图片、字符串、颜色、尺寸等资源，以及如何支持多语言。

**覆盖内容**：
- 资源文件的组织方式
- 图片资源（不同分辨率适配）
- 字符串资源（多语言支持）
- 颜色与主题
- 屏幕适配（不同尺寸、密度的设备）
- 夜间模式 / 深色模式

**跨端对应**：

| 概念 | Android | iOS | 鸿蒙 |
|------|---------|-----|------|
| 资源目录 | res/ (drawable / values / layout) | Assets.xcassets / .strings | resources/ |
| 图片适配 | mdpi / hdpi / xhdpi | @1x / @2x / @3x | 资源限定词 |
| 字符串 | strings.xml | Localizable.strings | string.json / resource |
| 颜色主题 | colors.xml + themes.xml | Asset Catalog Color / UIColor | Theme |
| 深色模式 | values-night/ | Dark Appearance | darkMode |
