# 公共概念层 -- 前端方向

> 本文件定义 React / Vue / 小程序 / H5 四端在前端方向中**共通的 10 个核心概念**。
>
> 用途：作为"AI 导师"教学的基准锚点 -- 学员在某一端学过的概念可以作为"第一层端类比"加速学习其他端；首学某概念时由 AI 用三层类比（端 / 背景 / 通识）讲清楚。
>
> **重要约束**：本文件仅适用于**前端方向**。客户端方向的概念定义见 `公共概念层-客户端.md`。两套概念层互不串通：前端端中的"组件模型"与客户端端中的"UI 组件体系"虽然在阶段中位置相同，但属于独立维护的两个概念。

## 1. 组件模型与 JSX/模板

**核心问题**：UI 是怎么用代码描述出来的？组件如何嵌套和复用？

**各端表现**：
- **React**：函数组件 + JSX，UI 即一个返回 JSX 的函数；props/children 传值
- **Vue**：单文件组件（SFC）`.vue` 文件，`<template>` + `<script setup>` + `<style scoped>`；props/slots 传值
- **小程序**：`.wxml`（微信）/ `.axml`（支付宝）等模板语言，配合 `.js` 逻辑文件；自定义组件通过 `Component()` 注册
- **H5（原生 / Web Components）**：原生 DOM API 或 Custom Elements；模板字符串或 `innerHTML`

**关键差异**：
- 声明式（React/Vue）vs 模板语言（小程序）vs 命令式（原生 H5）
- 单向数据流（React 的 props）vs 双向绑定（Vue 的 v-model）
- 组件作用域：JSX 是 JS 表达式 vs 模板有专门语法

## 2. 组件生命周期与 Hooks

**核心问题**：组件从挂载、更新到卸载的过程中，开发者在哪些时机可以介入？

**各端表现**：
- **React**：函数组件用 `useEffect`（替代了类组件的 `componentDidMount` / `componentDidUpdate` / `componentWillUnmount`）；依赖数组控制触发时机
- **Vue**：`onMounted` / `onUpdated` / `onUnmounted`（Composition API）或 `mounted` / `updated` / `unmounted`（Options API）
- **小程序**：页面级 `onLoad` / `onShow` / `onReady` / `onHide` / `onUnload`；组件级 `attached` / `ready` / `detached`
- **H5**：Custom Elements 的 `connectedCallback` / `disconnectedCallback` / `attributeChangedCallback`

**关键差异**：
- React 的 Hooks 把生命周期和状态交错耦合（同一 Hook 既管订阅也管清理）
- 小程序额外有"页面"和"组件"两种生命周期，且页面有 `onShow` 这种"路由进入时"的概念

---

## 3. 事件系统与合成事件

**核心问题**：用户的点击、输入等操作如何被代码捕获和处理？

**各端表现**：
- **React**：合成事件（SyntheticEvent），统一封装、跨浏览器一致；`onClick` 等命名为驼峰
- **Vue**：模板上 `@click="handler"`；支持事件修饰符 `.stop` / `.prevent` / `.once`
- **小程序**：`bindtap` / `catchtap`（catch 阻止冒泡）；事件参数通过 `dataset` 或 `event.currentTarget.dataset` 传递
- **H5**：原生 DOM 事件，通过 `addEventListener` 注册；冒泡和捕获阶段需开发者自己理解

**关键差异**：
- React 的合成事件是事件委托到 root，避免逐元素绑定
- Vue 修饰符让模板更声明式
- 小程序事件参数传递必须走 `data-*` 属性，不能像 React/Vue 一样直接 inline 传参

## 4. 路由

**核心问题**：单页应用如何在不刷新整页的情况下切换"页面"？参数如何传递？

**各端表现**：
- **React**：React Router（`<BrowserRouter>` / `<Routes>` / `<Route>`），`useNavigate` 触发跳转，`useParams` / `useSearchParams` 取参
- **Vue**：Vue Router（`createRouter` / `<router-view>`），`useRoute` / `useRouter`，`router.push({ name, params, query })`
- **小程序**：原生 API `wx.navigateTo` / `wx.redirectTo` / `wx.switchTab` / `wx.navigateBack`；URL 风格 `/pages/x/x?id=1`，参数取自页面 `onLoad(options)`
- **H5**：`window.location` 或 `history.pushState`；hash 路由用 `location.hash`

**关键差异**：
- 历史栈：React/Vue 路由基于 History API；小程序有自己的页面栈（最多 10 层）
- 参数传递：Web 端基于 URL；小程序参数大小受 URL 长度限制（推荐用 `EventChannel` 传复杂对象）

## 5. 状态管理

**核心问题**：跨组件共享的状态放在哪里？组件间如何通信？

**各端表现**：
- **React**：`useState`（组件内）/ `useReducer`（局部复杂）/ `useContext`（跨层）/ Redux / Zustand / Jotai（全局）
- **Vue**：`ref` / `reactive`（组件内）/ `provide` / `inject`（跨层）/ Pinia / Vuex（全局）
- **小程序**：页面内 `setData` / `data`；全局用 `getApp()` 的 `globalData` 或 MobX-miniprogram
- **H5**：组件间通信用事件（CustomEvent）或外部状态库

**关键差异**：
- 不可变（React 强制返回新对象）vs 可变（Vue 直接改 reactive 对象）
- 数据驱动 UI 的细粒度：React 的 setState 触发组件重渲染；Vue 的响应式追踪只重渲染依赖的部分

---

## 6. 网络请求

**核心问题**：如何向服务端获取数据，如何处理 loading / error / success 三种状态？

**各端表现**：
- **React**：`fetch` / `axios`，配合 `useEffect`；React Query / SWR 做缓存和状态管理
- **Vue**：`fetch` / `axios`，配合 `onMounted`；Vue Query / Pinia + composable
- **小程序**：原生 API `wx.request`（不支持流式、不支持 cookies），有并发数限制（默认 10）
- **H5**：原生 `fetch` / `XMLHttpRequest`

**关键差异**：
- 小程序的 `wx.request` 必须在合法域名白名单中（管理后台配置）
- 跨域：Web 端有 CORS 约束，小程序不存在浏览器同源策略但有合法域名约束
- 拦截器：axios 内置；`wx.request` 需自行封装

---

## 7. 浏览器存储

**核心问题**：如何在客户端持久化少量数据？

**各端表现**：
- **React / Vue / H5**：`localStorage`（同步、字符串、~5MB）/ `sessionStorage`（标签页生命周期）/ `IndexedDB`（异步、结构化、大容量）/ `Cookie`（伴随请求发送）
- **小程序**：`wx.setStorageSync` / `wx.getStorageSync`（同步、单 key 1MB、总量 10MB）；异步版本 `wx.setStorage` 等

**关键差异**：
- 小程序无 `localStorage` API，但语义类似（同步、key-value）
- IndexedDB 适合 Web 端大量数据；小程序无对应物
- Cookie 与 HTTP 强相关，小程序网络请求不自动带 cookies

---

## 8. 异步与 Promise

**核心问题**：JS 单线程下，如何处理并发的、有先后依赖的异步任务？

**各端表现**：
- **React / Vue / H5**：原生 `Promise` / `async-await`；`Promise.all` / `Promise.race` / `Promise.allSettled`
- **小程序**：早期 API 多为回调风格（`success` / `fail` / `complete`）；现代版本支持 Promise（`wx.getStorage()` 不传 success 时返回 Promise）
- **Web Worker**：CPU 密集任务移出主线程（H5 可用，小程序有自己的 Worker）

**关键差异**：
- 微任务 vs 宏任务的执行顺序（事件循环常考点）
- React 18+ 的 `useTransition` / `Suspense` 引入了新型异步原语
- 小程序的 Worker 有特殊限制（不能用 DOM、不能用 wx.* 大部分 API）

---

## 9. 性能与渲染优化

**核心问题**：如何让页面又快又流畅？哪些操作会触发不必要的重渲染或重绘重排？

**各端表现**：
- **React**：`React.memo` / `useMemo` / `useCallback` / 虚拟列表（react-window）/ Code Splitting + lazy
- **Vue**：`v-memo` / `KeepAlive` / 虚拟列表（vue-virtual-scroller）/ 异步组件
- **小程序**：`setData` 性能（diff 体积）/ 长列表用 `recycle-view` / 分包加载
- **H5**：避免重排（reflow）/ 重绘（repaint）/ requestAnimationFrame / will-change

**关键概念**：
- 虚拟 DOM 的 diff 算法（React fiber、Vue 的 patch）
- 浏览器渲染流水线：Parse → Style → Layout → Paint → Composite
- 关键渲染路径（Critical Rendering Path）
- 懒加载（图片、组件、路由）

---

## 10. 工程化（构建 / 打包 / 部署）

**核心问题**：源代码如何变成最终交付的产物？怎么保证开发体验和上线质量？

**各端表现**：
- **React / Vue / H5**：Webpack / Vite / Rspack / Turbopack；npm scripts；CI/CD（GitLab CI / Jenkins）
- **小程序**：原生开发者工具（自带构建）；Taro / uni-app（多端编译）
- **构建产物**：JS bundle（chunk splitting）/ CSS（提取、压缩）/ 静态资源（hash 命名、CDN）

**关键概念**：
- 模块化：CommonJS / ESM / UMD
- Tree shaking、Code splitting、Dynamic import
- 环境变量与多环境配置（dev / staging / prod）
- Source map 与线上调试
- 性能监控与错误上报（Sentry / 自建 SDK）
- 灰度发布与回滚

## 跨端类比说明

四端在每个公共概念上**有相似机制但实现细节不同**。AI 在教学时的策略：

1. **学员声明已掌握某端**（如 React） → 学习其他端时，该端作为"第一层端类比"锚点
2. **新学某概念时** → 用学员熟悉的端作为类比起点，说明差异
3. **示例**：学员已会 React，学 Vue 的状态管理 → "Vue 的 `ref` 类似 React 的 `useState`，但不需要解构 setter；直接 `.value =` 修改即可"

类比之后**必须指出关键差异**，避免学员误以为"完全一样"。
