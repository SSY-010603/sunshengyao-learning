# 三端 Hello World Demo 执行步骤总结

## 一、iOS

### 环境安装

macOS 自带 Xcode 和命令行工具，无需额外安装。你的机器已有 Xcode 26.4。

额外装了 `xcodegen`（用 Homebrew），用来从 `project.yml` 生成 `.xcodeproj`：

```bash
brew install xcodegen
```

### 构建与运行

```bash
cd demos/ios/HelloWorld

# 1. 从 project.yml 生成 Xcode 项目
xcodegen generate

# 2. 编译（模拟器版本）
xcodebuild -project HelloWorld.xcodeproj -scheme HelloWorld \
  -sdk iphonesimulator -configuration Debug -derivedDataPath build build

# 3. 启动模拟器
xcrun simctl boot <模拟器UUID>   # 可用 xcrun simctl list devices available 查 UUID

# 4. 安装 .app 到模拟器
xcrun simctl install <模拟器UUID> build/Build/Products/Debug-iphonesimulator/HelloWorld.app

# 5. 启动 App
xcrun simctl launch <模拟器UUID> com.example.HelloWorld

# 6. 打开模拟器窗口
open -a Simulator
```

### 实际踩坑

- 第一次构建失败：缺少 `Info.plist`。解决：在 `project.yml` 里加 `GENERATE_INFOPLIST_FILE: YES`
- 修复后 **BUILD SUCCEEDED**，模拟器上运行正常

### 关键文件

```
demos/ios/HelloWorld/
├── project.yml              ← xcodegen 的项目描述文件
├── HelloWorld.xcodeproj/    ← 自动生成，不要手动编辑
└── HelloWorld/
    ├── AppDelegate.swift    ← App 入口
    └── ViewController.swift ← 首页
```

## 二、Android

### 环境安装

需要三个组件，全部用 Homebrew：

```bash
# 1. JDK 17（Android 构建要求）
brew install openjdk@17
# JDK 安装位置：/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home

# 2. Android 命令行工具（包含 sdkmanager）
brew install android-commandlinetools

# 3. SDK 组件（用 sdkmanager 安装）
export JAVA_HOME=/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home
export ANDROID_HOME=$HOME/Library/Android/sdk
mkdir -p $ANDROID_HOME
yes | sdkmanager --sdk_root=$ANDROID_HOME \
  "platforms;android-34" "build-tools;34.0.0" "platform-tools"

# 4. Gradle（用来生成 wrapper，之后就不需要了）
brew install gradle
```

### 构建与运行

```bash
cd demos/android/HelloWorld

# 环境变量（每次构建前设置，或写入 ~/.zshrc 永久生效）
export JAVA_HOME=/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home
export ANDROID_HOME=$HOME/Library/Android/sdk
export ANDROID_SDK_ROOT=$HOME/Library/Android/sdk

# 1. 编译
./gradlew assembleDebug

# 2. APK 位置
# app/build/outputs/apk/debug/app-debug.apk

# 3. 安装到模拟器/真机（需先启动模拟器或连接真机）
adb install app/build/outputs/apk/debug/app-debug.apk

# 4. 启动 App
adb shell am start -n com.example.helloworld/.MainActivity
```

### 实际踩坑

- Kotlin 1.9.20 + Gradle 8.5 编译失败（`HasConvention` 兼容问题）。解决：升级到 Kotlin 2.0.21 + Gradle 8.11.1 + AGP 8.7.3
- Kotlin daemon 编译时报 `NoSuchFileException`。解决：`rm -rf .gradle app/build build` 清理后重编
- 修复后 **BUILD SUCCEEDED**，APK 5.5MB

### 关键文件

```
demos/android/HelloWorld/
├── build.gradle.kts           ← 根构建文件
├── settings.gradle.kts        ← 仓库和模块配置
├── gradle.properties          ← Gradle 参数

```

## 三、鸿蒙

### 环境安装

**只能通过 DevEco Studio 安装**，没有命令行独立安装方式。

1. 下载 DevEco Studio：https://developer.huawei.com/consumer/cn/deveco-studio/
2. 安装后打开，进入 SDK Manager 安装 HarmonyOS SDK（API 12+）
3. DevEco Studio 自带 ohpm、hvigor、Ark 编译器、模拟器，无需额外安装

### 构建与运行

**不能纯命令行构建**。`@ohos/hvigor-ohos-plugin` 等构建插件只在 DevEco Studio 内部仓库可用，公共 ohpm 仓库拉不到。

必须通过 DevEco Studio 操作：

```
1. 打开 DevEco Studio → Create Project
2. 选择 Empty Ability → Next
3. 填写项目信息 → Finish
4. 等 hvigor 同步完成
5. 替换源码：
   - entry/src/main/ets/pages/Index.ets
   - entry/src/main/ets/entryability/EntryAbility.ets
6. 点 ▶️ Run，选择模拟器或真机
```

### 实际踩坑

- ohpm 公共仓库找不到 `@ohos/hvigor-ohos-plugin`，尝试加华为私有仓库也不行
- 结论：鸿蒙项目必须通过 DevEco Studio 创建和构建，命令行方案目前不可行

### 源码文件（需替换到 DevEco 创建的项目中）

```
demos/harmony/HelloWorld/entry/src/main/
├── ets/
│   ├── entryability/EntryAbility.ets  ← Ability 入口
│   └── pages/Index.ets               ← 首页（@State + 按钮 + Column）
├── module.json5                       ← 模块配置
└── resources/base/
    ├── element/string.json, color.json
    └── profile/main_pages.json
```
