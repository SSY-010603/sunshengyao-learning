Kwai Tutor Starter

快手主站技术部 1-on-1 AI 学习导师 · 基于 Bloom 掌握学习法 · 支持客户端与前端两个方向

这是一个 学习仓库模板，配合 AI agent（推荐 MyFlicker）为你提供个性化的学习路径：
客户端方向：Android / iOS / 鸿蒙 三端
前端方向：React / Vue / 小程序 / H5 四端
⚠️ 两个方向互不串通：同方向内跨端走对照模式；跨方向从入门走起，已有另一方向经验仅作为背景锦标。

🤖 AI 驱动 📐 三层类比 🔄 反馈迭代 📊 阶段进度 🌍 双方向 7 端

✨ 它是什么

  • 不是 视频课程、不是文档教程、不是 RAG 知识库
  • 是 一个 AI 导师的执行规范 + 你个人的学习记录仓库
  • AI 会根据你的背景、节奏、反馈，为你一篇一篇生成学习文章；你写反馈，它读反馈，迭代教学

核心特性

特性说明
三层类比教学用你已知的知识（其他端 / 编程语言 / CS 通识）做类比锚点
双模学习路径零基础入门 + 跨端迁移，自动判断
反馈驱动迭代你不写反馈，AI 不会推进；写得越多，下一篇越精准
导师可视化全流程在 git 仓库内，你的导师随时跟进

🚀 学员快速开始（5 步）

1

clone starter 仓库到临时位置

# 把 starter clone 到任意临时位置（不要直接在这里工作）
git clone <starter-url> /tmp/Kwai-tutor-starter
cd /tmp/Kwai-tutor-starter
2

跑 bootstrap 脚本，生成你自己的学习仓库

bash scripts/bootstrap.sh <你的邮箱前缀>
# 例如：
#   bash scripts/bootstrap.sh zhangsan03

脚本会自动：

  • 校验你的环境（git 已安装、user.name / user.email 已配置）
  • 在 ../{你的邮箱前缀}-tutor-learning/ 下复制 starter 内容（不带 .git，干净独立）
  • 初始化新 git 仓库并完成首次 commit -m "init"
  • 输出后续操作引导
⚠️
校验失败时脚本会停下并给出明确错误提示（缺参数、git 未配置、目标目录已存在等），按提示修复即可。
3

在 GitLab 创建空仓库并推送

这一步创建新仓库是为了团队能更好地同步进度，可以一个团队共用一个仓库或单人单仓（建议导师创建），以下为建议示例：

进入内网 GitLab：

  • 新建仓库（建议导师创建），命名建议：{邮箱前缀}-tutor-learning
  • 不要勾选 "Initialize repository with a README"

然后关联 remote 并推送：

cd /tmp/{邮箱前缀}-tutor-learning   # 注意：bootstrap 把它放在 starter 的同级目录
git remote add origin <你的 GitLab 仓库 url>
git branch -M main
git push -u origin main
4

启动学习

  • 把仓库 clone 到你日常工作的目录
  • 在仓库根目录启动 MyFlicker（AGENTS.md 会被 AI 助手自动加载）
  • 在 AI 对话框中输入：

我要开始学习

AI 会按 AGENTS.md 规范走完整流程：

  1. 先问你画像首问：想学哪个方向（客户端 / 前端 / 都想学）
  2. 追问已有开发经验（开放回答，AI 自动归类为客户端/前端/其他项目经验）
  3. 如果你声明已熟悉当前方向某端，AI 会追问粗粒度档位（A/B/C/D），并预填该端的公共概念掌握情况和阶段进度
  4. AI 会把预填结果展示给你 review —— 如果某个概念你其实不太熟，告诉 AI，它会改回 ⚪
  5. 在 students/{你的名字}/ 下创建 profile.md
  6. 让你选当前方向的第一端 → 生成 01.md
💡
不熟悉就大胆说不熟。AI 不会因此小看你 —— 这只决定了下一篇文章从哪里开始，宁可慢一点把基础补齐。
⚠️
选了"都想学"？你要先选一个主方向作为起点，二方向在主方向完成阶段 1 后才能开启（避免两边都不扎实）。

📚 日常学习节奏

读 → 写反馈 → commit → "继续"

具体来说：

  1. AI 生成 students/{你的名字}/{方向}/{端}/01.md（如 students/zhangsan/客户端/Android/01.md 或 students/zhangsan/前端/React/01.md）
  2. 你打开文件，仔细阅读正文
  3. 滚动到文末反馈区，填写：
    • 【我理解到的核心】（必填，2-3 句话复述本文要点）
    • 还有疑问的地方（可选）
    • 想深入的方向（可选）
    • 联想到的（可选）
  4. git add . && git commit -m "feat: complete Android/01" && git push
  5. 回到 AI 对话，说"继续"或"下一篇"，AI 会读你的反馈，生成 02.md
⚠️
重要约束：如果你没填写"我理解到的核心"，AI 会拒绝生成下一篇。这是 Bloom 掌握学习法的核心机制 —— 没有真正的内化，就没有下一步。

🎯 学习阶段说明

整个学习路径按 3 个阶段 推进。两个方向各有三个阶段，命名相同但涵盖概念不同。每篇 md 末尾都会自动显示你当前的阶段进度。

客户端方向

阶段 1

入门 Onboarding

UI 组件 · 生命周期 · 事件 · 导航
独立完成纯 UI 改造；可作为复杂需求里的"页面骨架"贡献者
阶段 2

进阶 Independent

状态管理 · 网络 · 持久化 · 异步
独立承接列表页、详情页、表单、登录注册等带数据流转的常规业务需求
阶段 3

完备 Production-Ready

权限 · 资源/国际化 · 综合训练
交付符合上线标准的功能；具备该端正式 owner 资格

前端方向

阶段 1

入门 Onboarding

组件模型与 JSX/模板 · 生命周期与 Hooks · 事件 · 路由
独立完成静态页面、营销活动落地页、设计稿还原
阶段 2

进阶 Independent

状态管理 · 网络请求 · 浏览器存储 · 异步与 Promise
独立承接列表页、详情页、表单、本地缓存等常规业务
阶段 3

完备 Production-Ready

性能与渲染优化 · 工程化 · 综合训练
交付含性能调优、可监控可回滚的上线需求；具备该端正式 owner 资格

阶段判定规则

  • 该阶段所有概念都 ✅ → 阶段已完成
  • 至少 1 个 ✅ 或 ⏳ → 阶段进行中
  • 全部 ⚪ → 阶段未开始

学员视角

每篇 md 末尾的"📊 学习进度"小节会显示：

  • 当前阶段（如"阶段 2 进阶"）
  • 阶段内已完成 / 总概念数
  • 当前能力等级（业务化描述，如"可承接 UI 改造类需求；正在解锁数据流转能力"）
  • 下一里程碑

导师视角

通过 students/{学员}/profile.md 的"阶段进度"小节，一眼看出学员目前能承接什么类型的需求。

同方向内跨端学员（如 Android → iOS）通常通过"对照模式"在 2-4 周内完成全部 3 个阶段。跨方向（如客户端 → 前端）需从阶段 1 重新走起，另一方向经验仅作为背景锦标。

✍️ 反馈写作指南

必填项：我理解到的核心

不是抄文章原话，而是用你自己的话复述。建议 2-3 句，覆盖：

  • 这篇文章的核心概念是什么
  • 它和你已经知道的东西怎么联系
  • 关键的 1-2 个要点
✅
好例子： Activity 是 Android 单个屏幕的载体，类似 iOS 的 UIViewController 或者 Web 的一个页面。它有 7 个生命周期回调，最关键的是 onCreate（初始化）和 onDestroy（清理资源），中间的 onPause/onResume 处理可见性。
❌
差例子： 学完了，理解了。

可选项：还有疑问的地方

写下卡点，AI 下一篇会优先解决。例如：

不太清楚为什么 onPause 和 onStop 是分开的，什么时候会只触发 onPause 不触发 onStop？

可选项：想深入的方向

如果对某个点特别感兴趣，可以引导后续节奏：

想多看几个 Fragment 和 Activity 通信的实战案例。

可选项：联想到的

帮 AI 找到更精准的类比锚点：

这让我想到 React 的 componentDidMount 和 componentWillUnmount，和 onCreate/onDestroy 很像。

👨‍🏫 导师视角

如果你是导师，本仓库提供以下跟进姿势：

默认：仅浏览模式

  • 定期 git pull 学员仓库（或在 GitLab 网页上查看）
  • 看 commit log 了解学员节奏
  • 看 students/{学员}/profile.md 了解掌握情况
  • 看每篇文章的反馈区了解学员真实理解程度

可选：在仓库开 Issue

如果你对某篇文章有疑问、想补充观点、或想引导学员关注某个方向：

  • 在学员仓库新建 Issue
  • 在标题写明"对应文章：students/zhangsan/Android/03.md"
  • 学员看到 Issue 后会在下一次对话中告诉 AI："导师在 issue #3 中提到 xxx，下篇请展开讨论"

不需要做的

  • ❌ 直接修改学员的 md 文件（会破坏 AI 的反馈循环）
  • ❌ 提 PR / merge request
  • ❌ 在 md 里写批注

📁 目录约定

your-tutor-learning/
├── AGENTS.md                       ← AI 导师执行规范（不要修改）
├── 公共概念层-客户端.md             ← 客户端三端共通概念（不要修改）
├── 公共概念层-前端.md               ← 前端四端共通概念（不要修改）
├── README.md                       ← 本文件
├── .gitignore
└── students/
    └── {你的名字}/                  ← AI 自动创建你的目录
        ├── profile.md              ← 你的画像、方向、进度
        ├── 客户端/                  ← 选了客户端才会创建
        │   ├── Android/
        │   │   ├── 01.md
        │   │   └── 02.md
        │   ├── iOS/
        │   └── 鸿蒙/
        └── 前端/                    ← 选了前端才会创建
            ├── React/
            ├── Vue/
            ├── 小程序/
            └── H5/

目录命名约定

  • 学员目录建议用邮箱前缀：students/zhangsan03/
  • 方向目录固定：客户端 / 前端（中文）
  • 端目录固定：Android / iOS / 鸿蒙 / React / Vue / 小程序 / H5
  • 文章固定 2 位数字：01.md、02.md、10.md
💡
注意：你的学习仓库由 bootstrap.sh 派生而来，不会包含 starter 仓库的 scripts/ 目录。它只属于 starter 本身。

❓ FAQ

Q1：starter 仓库的规范文档（AGENTS.md / 公共概念层-客户端.md / 公共概念层-前端.md）更新了，我怎么同步？

第一版采用手动同步：

  1. 关注内网 GitLab 上 Kwai-tutor-starter 的 commit history
  2. 看到有更新时，复制对应文件覆盖到自己仓库
  3. commit 时备注 "chore: sync spec from starter @ {commit-hash}"

如果后续迭代频繁，会提供同步脚本。

Q2：我想换一端学，或者换个方向学，怎么操作？

同方向内换端（如 Android → iOS）：直接在对话里告诉 AI：

我想开始学 iOS 了

AI 会读 profile.md，自动判断用入门模式还是跨端过渡模式（如果你 Android 已掌握一些公共概念，会在 iOS 端使用对照模式加速）。

跨方向（如客户端 → 前端）：

  • 初始选了"都想学"：在主方向完成阶段 1 后，说"我想开始学 {二方向}"，AI 会创建二方向目录并从阶段 1 走起
  • 初始只选了一个方向但后面想加：可以让 AI 帮你更新 profile.md 的开发方向字段为"客户端 + 前端待开启"，仍需主方向完成阶段 1 后才能开启二方向
⚠️
跨方向学习不会拿另一方向的 ✅ 作为预填。例如你是 Android 高手，学 React 时 React 方向依然从零起步，Android 的背景仅作为类比锦标加速理解。
Q3：我的 md 文件越来越长，反馈区找不到了？

这是正常的。建议：

  • 一篇文章的字数控制在 2000-4000 字（AI 默认会控制）
  • 单个公共概念如果太大（如"UI 组件体系"），AI 会自动拆成多篇（01.md、02.md、03.md）
  • 反馈区永远在文件最末尾，按 Cmd+End / Ctrl+End 直接跳到底部
Q4：我中断了一段时间，回来怎么继续？

直接打开 AI 对话说"继续"。AI 会读 profile.md 中的"最近学习时间"，如果超过 30 天会建议你先复习关键概念，再进入下一篇。

Q5：我可以同时学两端吗？

同方向内两端（如 Android + iOS）：技术上可以，但不推荐。AI 会建议你先完成一端的至少 5 个核心概念，再开第二端，否则类比锚点不够强。

同时学两个方向（如客户端 + 前端）：不允许并行。需要先完成主方向阶段 1，才能开启二方向。这是为避免精力过度分散导致两边都不扎实。
